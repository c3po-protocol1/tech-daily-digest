## 1. 탄생 배경 — "네트워크 레이턴시가 거슬렸다"

저자의 동기가 매우 구체적이다. 기존에 **superwhisper**, **spokenly** 같은 음성 타이핑 도구를 8bitdo micro 게임패드와 연결해서 사용하고 있었는데, **클라우드 기반 음성 인식의 네트워크 레이턴시가 묘하게 거슬렸다**고 한다. 음성 입력 → 결과 출력까지의 지연이 수백 ms에 불과하더라도, 타이핑의 즉각적 피드백에 익숙한 사용자에게는 그 지연이 워크플로를 방해한다.

로컬 모델을 테스트하던 중 **Qwen3-ASR(Alibaba의 자동 음성 인식 모델)**을 발견하고, 클라우드 모델에 뒤지지 않는 성능에 빠릿빠릿한 속도를 확인한 것이 결정적 계기가 되었다. 이 모델을 로컬에서 구동하면서 기존 상용 앱과 동일한 UX를 제공하기 위해 vvrite를 만들었다.

## 2. 작동 방식 — 단순함의 힘

vvrite의 UX는 극도로 단순하다:

1. **핫키를 누른다** (기본: `Option + Space`) — 어떤 앱에서든 작동
2. **말한다** — 녹음 오버레이가 화면에 나타나며 오디오 레벨 바와 타이머를 표시
3. **핫키를 다시 누른다** — 녹음 중지
4. **텍스트가 자동으로 붙여넣어진다** — 현재 활성 텍스트 필드에 직접 삽입

`ESC`를 누르면 녹음을 취소할 수 있다. 메뉴 바에 조용히 상주하며, 설정은 메뉴 바 아이콘 클릭으로 접근한다.

**핵심: 모든 처리가 디바이스에서 이루어진다.** 음성 데이터가 Mac을 떠나지 않는다.

## 3. 기술 스택 — Apple Silicon의 AI 역량을 활용

| 구성 요소 | 기술 |
|---|---|
| **UI** | PyObjC (AppKit, Quartz) — 네이티브 macOS 통합 |
| **ASR 모델** | **Qwen3-ASR-1.7B-8bit** (Alibaba) |
| **추론 엔진** | **MLX** (Apple의 ML 프레임워크) — Apple Silicon GPU에서 직접 실행 |
| **오디오** | sounddevice + ffmpeg |
| **패키징** | PyInstaller → 서명/공증된 DMG |

### Qwen3-ASR — 왜 Whisper가 아닌가

이 프로젝트의 기술적 핵심 결정은 **OpenAI Whisper 대신 Qwen3-ASR을 선택**한 것이다. GeekNews 댓글에서 저자가 직접 밝힌 비교:

- **한국어 기준으로 Whisper보다 확실히 좋다** — Whisper는 영어에 최적화되어 있어 한국어 인식 정확도가 상대적으로 떨어진다
- **Whisper large v3 turbo와 비교해도 Qwen3-ASR이 한국어에서 우위** — 토론에서 직접 비교 경험을 공유
- **Moonshine이나 Parakeet과 비교 시** — Parakeet은 영어에 적합하지만, 한국어에서는 Qwen3-ASR이 최선

1.7B 파라미터의 8bit 양자화 모델로, **약 2GB 디스크 공간**만 필요하다. Apple Silicon의 Neural Engine과 GPU를 활용하여 실시간에 가까운 속도로 추론한다.

### MLX — Apple의 로컬 AI 프레임워크

**MLX**는 Apple이 개발한 ML 프레임워크로, Apple Silicon의 통합 메모리 아키텍처(Unified Memory Architecture)에 최적화되어 있다. CPU와 GPU가 메모리를 공유하므로, 데이터를 GPU로 복사하는 오버헤드가 없다. 이는 NVIDIA GPU 기반 추론 대비 메모리 효율에서 큰 장점을 가지며, 노트북에서도 실용적 수준의 AI 추론을 가능하게 한다.

MLX 기반 오디오 처리 라이브러리 `mlx-audio`를 사용하여 Qwen3-ASR 모델을 구동한다.

## 4. GeekNews 커뮤니티 반응 — 12개 댓글의 가치

GeekNews에서 12개 댓글이 달렸으며, 실사용 피드백이 활발했다. 주요 논점:

### 한국어 음성 인식의 현주소

한국어 음성 인식은 영어 대비 선택지가 제한적이다. 대부분의 오픈소스 ASR 모델은 영어에 최적화되어 있으며, 한국어 지원은 부차적이다. 저자가 "한국어 지원하는 좋은 로컬 모델이 없어서 직접 만들었다"고 밝힌 것이 이 현실을 보여준다.

댓글에서 한 사용자는 **안드로이드용 음성 명령 앱**을 개발 중인데, Qwen3-ASR이 좋지만 아직 모바일 최적화가 부족하여 **Whisper base를 한국어 데이터셋으로 파인튜닝**한 모델을 사용하고 있다고 밝혔다. 이는 로컬 AI의 플랫폼별 최적화 차이를 보여준다 — macOS + Apple Silicon에서는 MLX가 있어 1.7B 모델도 쉽게 구동하지만, 모바일에서는 여전히 더 작은 모델이 필요하다.

### 실사용 경험

- "써보니 빠르고 좋네요" — 즉각적 긍정 반응
- 한 사용자는 **모델 3개를 추가하고, 알림 소리 조절, 버그 수정** 등 커스터마이징을 진행하며 "입코딩 신나게 하겠습니다"라고 댓글
- "코드 열어보니 개발자 아닌 분이 진행하시는 프로젝트 같아서 더 응원하게 되네요" — AI 도구로 비개발자도 실용적 앱을 만들 수 있는 시대를 보여주는 코멘트
- "superpowers 흔적 남아있는 것도 보면서 '아 이렇게들 하시는구나...'" — 저자가 Claude Code의 superpowers 스킬을 사용하여 개발한 흔적이 코드에 남아있다는 관찰

### 기존 도구와의 차이

"handy(유사 도구)와 차이점이 무엇인가"라는 질문에 대한 저자의 답변이 핵심을 찌른다:

> "이런 류의 앱들이 UX는 거의 비슷할 거예요. **결국 모델 성능이죠.** 영어만 쓰실 거면 handy에서 지원하는 parakeet 정도면 충분한데요, 한국어 지원은 경험상 Qwen3-ASR이 제일 좋은 옵션으로 다가왔습니다. 근데 지원하는 곳이 없어서 직접 만들었어요."

음성 타이핑 앱의 **차별화 포인트는 UI가 아니라 ASR 모델**이라는 것이다. 같은 UX라도 어떤 모델을 사용하느냐에 따라 사용자 경험이 극적으로 달라진다.

## 5. "비개발자가 AI로 앱을 만들다" — 숨겨진 스토리

GeekNews 댓글에서 가장 의미심장한 관찰은 **"코드 열어보니 개발자 아닌 분이 진행하시는 프로젝트 같다"**는 것이다. 코드에 Claude Code의 superpowers 스킬 흔적이 남아 있다는 것은, 저자가 **AI 코딩 에이전트를 활용하여 이 앱을 개발**했음을 시사한다.

이것이 의미하는 바는 크다. 음성 타이핑 앱은 macOS의 네이티브 API(AppKit, Quartz), 오디오 처리(sounddevice, ffmpeg), ML 추론(MLX), 글로벌 핫키, 접근성 권한 등 다양한 기술 영역을 다뤄야 하는 상당히 복잡한 프로젝트이다. 이런 프로젝트를 **AI 에이전트의 도움으로 비전문 개발자가 실용적 수준으로 완성**한 것은, AI 코딩 도구의 현재 역량을 보여주는 실제 사례이다.

18포인트와 12개 댓글이라는 GeekNews의 반응도, 결과물이 "AI로 만든 장난감"이 아니라 **실제로 유용한 도구**임을 증명한다.

## 6. 로컬 AI 스택의 현재 — macOS에서의 가능성

vvrite는 **macOS + Apple Silicon에서 구동 가능한 로컬 AI 스택**의 구체적 사례이다. 현재 macOS에서 클라우드 없이 로컬로 실행 가능한 AI 도구 생태계를 정리하면:

| 영역 | 도구/모델 | 프레임워크 |
|---|---|---|
| **음성 인식(ASR)** | Qwen3-ASR, Whisper | MLX, mlx-audio |
| **텍스트 생성(LLM)** | Llama, Mistral, Qwen | Ollama, MLX |
| **이미지 생성** | Stable Diffusion, FLUX | MLX, Draw Things |
| **음성 합성(TTS)** | Kokoro, Piper | MLX |
| **임베딩/검색** | nomic-embed, bge | Ollama |

이 스택이 모두 **Apple Silicon의 통합 메모리와 MLX**를 기반으로 작동하며, 클라우드 API 비용 없이, 프라이버시를 보장하면서, 오프라인에서도 사용 가능하다. vvrite는 이 중 ASR 영역의 구현체이다.

## 7. Soo에게 의미 있는 이유

**첫째, "비개발자도 AI로 실용적 앱을 만들 수 있다"는 강력한 사례.** AInD 컨설팅에서 "AI 코딩 도구가 실제로 어느 수준까지 가능한가?"라는 질문에 대한 답변으로, vvrite는 복잡한 네이티브 앱도 AI 에이전트와 함께 만들 수 있음을 보여주는 실증 사례이다.

**둘째, 온디바이스 AI의 실용성 증명.** AInD 컨설팅에서 "클라우드에 데이터를 보내기 싫다"는 기업의 요구에 대해, "Apple Silicon에서 로컬로 충분한 품질의 AI를 구동할 수 있다"는 것을 vvrite가 증명한다. 음성 데이터는 특히 프라이버시에 민감하므로(미팅 내용, 의료 기록 등), 온디바이스 처리의 가치가 높다.

**셋째, 한국어 AI의 현실적 한계와 해법.** 한국 기업을 대상으로 컨설팅할 때 "한국어 지원은 어떤가요?"는 반드시 나오는 질문이다. vvrite의 Qwen3-ASR 선택 과정은 한국어 ASR의 현재 옵션과 최선의 선택지를 보여주는 실전 레퍼런스이다. Whisper(OpenAI) < Qwen3-ASR(Alibaba)이라는 한국어 성능 비교는 모델 선택 시 참고할 수 있다.

**넷째, 음성→텍스트 파이프라인의 기업 활용.** 음성 타이핑은 개인 생산성 도구이지만, 같은 기술을 기업에 적용하면 **미팅 자동 기록, 고객 상담 분석, 현장 보고서 음성 입력** 등의 유스케이스가 된다. 온디바이스 처리라는 특성은 데이터 보안 요구사항이 높은 기업에 특히 적합하다.
