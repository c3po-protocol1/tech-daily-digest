## 1. 무엇이 출시되었나

Anthropic이 **Claude Code Desktop**과 새로운 제품 **Claude Cowork**을 통해, Claude가 실제 컴퓨터의 **마우스, 키보드, 화면을 직접 제어**하는 기능을 공식 출시했다. 여기에 **Dispatch**를 결합하면, 사용자가 자리를 비운 상태에서도 Claude가 원격으로 컴퓨터를 조작하도록 지시할 수 있다.

핵심 제품 3가지:
- **Claude Cowork** — 목표를 주면 Claude가 컴퓨터, 로컬 파일, 애플리케이션을 사용하여 완성된 결과물을 반환하는 자율 에이전트
- **Claude Code Desktop** — 개발자용 CLI 환경에서 컴퓨터 제어 기능을 활용할 수 있는 데스크톱 앱
- **Dispatch** — 원격에서 Claude에게 컴퓨터 작업을 지시하는 비동기 태스크 위임 시스템

macOS에 먼저 출시되었으며, Windows는 수 주 내 지원 예정이다.

## 2. Computer Use의 기술적 원리

### 화면 읽기 (Vision)
Claude는 **스크린샷을 캡처하여 화면의 현재 상태를 "본다."** 이것은 Claude의 멀티모달(비전) 능력을 활용한 것이다. 화면의 스크린샷을 분석하여 UI 요소(버튼, 텍스트 필드, 메뉴, 대화상자 등)를 식별하고, 각 요소의 위치와 상태를 파악한다.

### 마우스/키보드 조작 (Action)
화면 분석 결과를 바탕으로, Claude는 **마우스 이동, 클릭, 드래그, 키보드 입력, 단축키** 등의 명령을 생성한다. 이 명령은 macOS의 Accessibility API를 통해 실제 시스템 이벤트로 변환되어, 어떤 GUI 애플리케이션이든 조작할 수 있다.

### 반복 루프 (Observe → Think → Act)
전체 과정은 반복적이다:
1. **Observe** — 화면 스크린샷 캡처 및 분석
2. **Think** — 현재 상태에서 목표를 달성하기 위한 다음 행동 결정
3. **Act** — 마우스/키보드 명령 실행
4. **Observe** — 결과 확인을 위해 다시 스크린샷 캡처
5. 반복...

이 루프는 Karpathy가 말한 **"Loopy" 패턴**의 구체적 구현이다 — 결과를 관찰하고, 평가하고, 다음 행동을 조정하는 반복적 자기 개선 루프.

## 3. Claude Cowork — "채팅이 아닌 결과물 중심"

Anthropic 공식 설명에서 가장 주목할 문장:

> **"대부분의 AI 도구는 프롬프트를 중심으로 설계되었다. Claude Cowork은 결과물(outcome)을 중심으로 설계되었다."**

Cowork의 철학은 기존 AI 채팅과 근본적으로 다르다. 채팅은 질문→답변의 단일 턴 상호작용이지만, Cowork은 **목표를 선언하면 Claude가 완성된 결과물을 반환**하는 태스크 위임 모델이다.

예시 시나리오:
- "이 10개 PDF에서 핵심 데이터를 추출하여 스프레드시트로 정리해줘" → Claude가 PDF를 열고, 데이터를 읽고, Excel/Sheets에 입력하고, 서식을 맞추고, 완성된 파일을 반환
- "이 프레젠테이션의 모든 차트를 최신 데이터로 업데이트해줘" → Claude가 PowerPoint를 열고, 데이터 소스에 접근하고, 차트를 수정하고, 저장
- "이 이메일에 대한 답장을 작성하고, 관련 문서를 첨부하여 보내줘" → Claude가 이메일 클라이언트를 열고, 답장을 작성하고, 파일 탐색기에서 문서를 찾아 첨부하고, 전송

**Anthropic이 명시적으로 "반복적이고, 지저분하고, 시간이 너무 오래 걸리는 작업"을 타겟**으로 하고 있다.

## 4. Dispatch — 자리를 비워도 일하는 Claude

Dispatch는 **비동기 태스크 위임**을 가능하게 한다. 사용자가 작업을 지시하고 자리를 비우면, Claude가 독립적으로 컴퓨터를 조작하여 태스크를 완수한다. 돌아왔을 때 결과물이 준비되어 있는 것이다.

이것은 에이전트 자율성(autonomy)의 새로운 수준이다. 기존에는 사용자가 에이전트의 행동을 실시간으로 감독해야 했지만, Dispatch는 **"감독 없는 자율 실행"**을 허용한다. Anthropic은 이것이 "Research Preview" 단계임을 강조하며, Computer Use가 아직 초기 단계라 Claude가 "사람보다 훨씬 느리고 신중하게 동작한다"고 밝히고 있다.

## 5. 기존 Computer Use(2024.10)와의 차이

Anthropic은 2024년 10월에 Computer Use를 처음 공개했다(Claude 3.5 Sonnet과 함께). 당시는 **API를 통한 개발자 전용 기능**이었으며, Docker 컨테이너 안에서 가상 데스크톱을 조작하는 방식이었다. 실제 사용자 컴퓨터가 아닌 격리된 환경에서만 작동했다.

이번 출시의 근본적 차이:

| 측면 | 2024.10 (초기) | 2026.03 (현재) |
|---|---|---|
| **대상** | 개발자 (API) | 모든 사용자 (GUI 앱) |
| **환경** | Docker 가상 데스크톱 | 실제 사용자 컴퓨터 (macOS) |
| **접근** | 격리된 샌드박스 | 실제 파일, 앱, 네트워크 접근 |
| **자율성** | 실시간 감독 필요 | Dispatch로 비동기 실행 가능 |
| **제품** | API 기능 | Claude Cowork (독립 제품) |
| **타겟 유스케이스** | 개발/테스트 자동화 | 지식 노동자의 반복 업무 |

이 전환은 **"기술 데모 → 상용 제품"**의 전환이며, Anthropic이 Computer Use를 핵심 제품 전략으로 격상시켰음을 보여준다.

## 6. 보안과 안전 — 가장 중요한 질문

에이전트가 실제 컴퓨터를 조작하는 것은 **강력하지만 위험**하다. 핵심 우려:

### 의도하지 않은 행동
Claude가 화면을 잘못 해석하여 잘못된 버튼을 클릭하거나, 잘못된 파일을 삭제하거나, 잘못된 이메일을 전송하는 시나리오가 가능하다. Anthropic이 "사람보다 훨씬 느리고 신중하게 동작한다"고 밝힌 것은 이 위험을 인식하고 속도를 의도적으로 제한한 것으로 보인다.

### 프롬프트 인젝션의 물리적 영향
웹 페이지에 숨겨진 프롬프트 인젝션이 에이전트의 행동을 조작하면, 그 영향이 **물리적 컴퓨터 조작**으로 이어질 수 있다. 텍스트 채팅에서의 프롬프트 인젝션은 잘못된 답변을 생성하는 수준이지만, Computer Use에서는 파일 삭제, 이메일 전송, 결제 등의 물리적 행동으로 이어질 수 있다.

### 접근 권한의 범위
Claude가 사용자의 컴퓨터에 로그인된 모든 서비스(이메일, 은행, 소셜 미디어 등)에 접근할 수 있다면, 그 권한의 범위와 제한이 핵심 설계 결정이 된다.

Anthropic은 에이전트 안전에 대한 프레임워크를 공식 발표한 바 있으며("Our framework for developing safe and trustworthy agents"), 이 원칙이 Cowork에 어떻게 적용되었는지가 향후 중요한 평가 포인트이다.

## 7. 산업적 임팩트 — RPA의 재정의

Computer Use는 기존 **RPA(Robotic Process Automation)** 산업을 근본적으로 위협한다.

| 측면 | 전통 RPA (UiPath 등) | Claude Computer Use |
|---|---|---|
| **설정** | UI 요소별 셀렉터 수동 설정 | AI가 자동으로 UI 이해 |
| **유연성** | UI 변경 시 스크립트 깨짐 | AI가 변경된 UI에 적응 |
| **개발** | RPA 전문가 필요 | 자연어로 태스크 지시 |
| **범위** | 사전 정의된 워크플로만 | 임의의 태스크 수행 가능 |
| **비용** | 라이선스 + 개발비 | AI 구독료만 |
| **한계** | 결정론적, 예측 가능 | 확률적, 때때로 예측 불가 |

전통 RPA의 가장 큰 고통점은 **UI 변경에 대한 취약성**이다. 버튼 위치가 바뀌거나 대화상자 텍스트가 변경되면 RPA 스크립트가 깨진다. Claude Computer Use는 화면을 "보고 이해"하므로, UI 변경에 자연스럽게 적응할 수 있다.

그러나 전통 RPA의 장점도 있다 — **결정론적 행동**이다. 같은 입력에 항상 같은 행동을 보장하며, 이는 금융이나 의료 같은 규제 환경에서 중요하다. Claude의 확률적 행동은 이런 환경에서 추가적 검증이 필요하다.

## 9. Soo에게 의미 있는 이유

### AInD 컨설팅에서의 핵심 임팩트

**첫째, "레거시 시스템 자동화"가 즉시 가능해진다.** 많은 기업이 API가 없는 레거시 시스템(오래된 ERP, 사내 관리 도구, 윈도우 전용 업무 앱 등)에 의존하고 있다. 기존에는 이런 시스템의 자동화가 RPA의 전유물이었지만, Claude Computer Use로 자연어 지시만으로 자동화가 가능해진다. 이것은 AInD 컨설팅의 **가장 즉각적이고 ROI가 명확한 유스케이스**이다.

**둘째, RPA 대체 시나리오.** 이미 RPA를 도입한 기업에게 "Claude Computer Use로 전환"을 제안하는 컨설팅이 가능하다. RPA의 유지보수 비용(UI 변경 시 스크립트 수정, 전문 인력 고용 등)을 AI 기반 접근으로 대폭 줄일 수 있다.

**셋째, "비개발자를 위한 AI"의 실현.** Cowork은 명시적으로 "가장 필요로 하는 사람들이 개발자가 아니라는 것"을 인식하고 설계되었다. 이는 AInD 컨설팅의 대상이 개발팀을 넘어 **영업, 마케팅, 인사, 재무 등 비기술 부서**로 확장될 수 있음을 의미한다.

**넷째, 보안 컨설팅의 새로운 영역.** 에이전트에게 컴퓨터 접근 권한을 부여하는 것의 보안 함의는 매우 크다. 기업의 보안팀과 협력하여 "어떤 범위까지 허용할 것인가", "어떤 행동에 승인을 요구할 것인가"를 설계하는 것이 컨설팅 딜리버리가 된다.

**다섯째, gstack + Computer Use의 결합.** gstack의 `/qa`가 이미 실제 브라우저(Playwright)에서 테스트를 수행하지만, Computer Use는 이를 **모든 GUI 앱**으로 확장한다. 네이티브 앱 디버깅, 데스크톱 앱 테스트 등 gstack이 커버하지 못하는 영역을 보완할 수 있다.
