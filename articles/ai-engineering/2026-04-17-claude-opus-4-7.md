- Anthropic이 Claude Opus 4.7을 정식 출시 — Opus 4.6의 **직접 업그레이드**
- 핵심 강점: **고난도 소프트웨어 엔지니어링 자율성**, 지시사항 문자 그대로 준수, 3배+ 고해상도 비전

## 1. 핵심 개선 사항

### 1.1 소프트웨어 엔지니어링 자율성

Opus 4.7의 가장 큰 변화는 **장시간 자율 코딩 작업에서의 신뢰성** 향상이다.

- 이전에 밀착 감독이 필요했던 가장 어려운 코딩 작업을 **자신 있게 맡길 수 있다**고 사용자 보고
- 복잡하고 장기 실행 태스크를 **엄밀하고 일관적으로** 처리
- 지시사항에 대한 **정밀한 주의**, 결과를 보고하기 전 **스스로 출력을 검증**하는 방법을 고안
- Vercel의 Joe Haddad: "시스템 코드에서 작업 시작 전 **증명(proofs)을 수행**하는 새로운 행동 — 이전 Claude 모델에서 본 적 없다"
- Shopify의 Ben Lafferty: "의미 없는 래퍼 함수와 폴백 스캐폴딩이 사라졌다. 코드를 작성하면서 **스스로 수정**한다"

### 1.2 지시사항 준수력 (Instruction Following)

**핵심 변화**: Opus 4.7은 지시사항을 **문자 그대로** 따른다.

- 이전 모델이 느슨하게 해석하거나 건너뛰던 지시를 **엄격하게 실행**
- **주의점**: 기존 프롬프트가 예상치 못한 결과를 낼 수 있음 — **프롬프트와 하네스 재조정 필요**
- Hex의 Caitlin Colgrove: "데이터가 없을 때 그럴듯하지만 틀린 폴백 대신 **데이터가 없다고 정확히 보고**한다"

### 1.3 멀티모달 — 3배 이상 고해상도 비전

- 최대 **2,576px 장변** (~3.75 메가픽셀) — 이전 모델 대비 **3배 이상**
- 모델 레벨 변경 (API 파라미터가 아님) — 사용자가 보내는 이미지가 자동으로 더 높은 해상도로 처리
- 고해상도가 불필요한 경우 전송 전 다운샘플링으로 토큰 절약 가능
- **활용 시나리오**: 컴퓨터 사용 에이전트의 밀집 스크린샷 판독, 복잡한 다이어그램에서 데이터 추출, 픽셀 퍼펙트 참조가 필요한 작업
- XBOW의 Oege de Moor: **시각 정밀도 벤치마크 98.5%** (Opus 4.6: 54.5%) — "가장 큰 Opus 페인 포인트가 사실상 사라졌다"

### 1.4 메모리 활용

- **파일시스템 기반 메모리**를 더 잘 활용
- 장기 멀티세션 작업에서 중요 메모를 기억하고, 새 작업에 활용
- 결과적으로 사전 컨텍스트(up-front context) 필요량 감소

### 1.5 실세계 업무 (Real-world Work)

- Finance Agent 평가에서 **SOTA** — 더 엄밀한 분석, 더 전문적인 프레젠테이션
- **GDPval-AA** (경제적 가치 있는 지식 작업 평가: 금융, 법률 등)에서 **SOTA**
- Harvey의 Niko Grupen: BigLaw Bench에서 90.9% — 양도 조항과 경영권 변경 조항을 **정확히 구별** (프런티어 모델의 역사적 약점)

## 3. 사이버 보안 세이프가드

### 3.1 Project Glasswing과의 연관

- Mythos Preview의 제한 출시에 따라, 덜 강력한 모델에서 **사이버 세이프가드를 먼저 테스트**하는 전략
- Opus 4.7이 **최초의 테스트 모델** — 사이버 역량은 Mythos Preview보다 약하며, 훈련 중 **차별적 역량 감소(differential capability reduction)** 실험
- **금지되거나 고위험 사이버보안 사용**을 자동 감지·차단하는 세이프가드 탑재
- 실전 배포에서의 학습이 **Mythos급 모델의 광범위한 출시** 목표에 기여

### 3.2 Cyber Verification Program

- 합법적 사이버보안 목적(취약점 연구, 침투 테스트, 레드팀)으로 Opus 4.7을 사용하려는 보안 전문가를 위한 **별도 검증 프로그램** 출시

## 4. 새로운 기능 및 도구

### 4.1 `xhigh` Effort 레벨

- `high`와 `max` 사이의 새로운 레벨
- 추론 깊이와 레이턴시 간의 **더 세밀한 제어**
- Claude Code 기본값이 전 플랜에서 **`xhigh`로 상향**
- 코딩/에이전틱 사용 시 `high` 또는 `xhigh`로 시작 권장

### 4.2 `/ultrareview` 명령어 (Claude Code)

- 변경사항을 읽고 **전용 리뷰 세션**에서 버그와 설계 이슈를 플래그
- 신중한 리뷰어가 잡을 만한 문제를 표면화
- Pro/Max 사용자에게 **3회 무료 ultrareview** 제공

### 4.3 Auto Mode 확대

- Max 사용자로 확대
- Claude가 **권한 결정을 대신** 수행 — 장시간 작업을 더 적은 중단으로 실행
- 모든 권한을 건너뛰는 것보다 **낮은 위험**으로 더 긴 자율 실행

### 4.4 Task Budgets (API, 퍼블릭 베타)

- 개발자가 Claude의 **토큰 소비를 가이드**하여 긴 실행에서 작업 우선순위 지정 가능

## 5. 마이그레이션 가이드: Opus 4.6 → 4.7

### 5.1 토크나이저 변경

- 업데이트된 토크나이저로 텍스트 처리 개선
- **트레이드오프**: 동일 입력이 **1.0~1.35배** 더 많은 토큰으로 매핑될 수 있음 (콘텐츠 유형에 따라)

### 5.2 높은 Effort에서 더 많은 출력 토큰

- 특히 에이전틱 세팅의 후반 턴에서 **더 많이 생각(thinking)**
- 어려운 문제에서 신뢰성 향상, 그러나 출력 토큰 증가
- 제어 방법: effort 파라미터, task budgets, 간결성 프롬프팅

### 5.3 순 효과

- Anthropic 내부 코딩 평가에서 **모든 effort 레벨에서 토큰 사용량 대비 성능 개선**
- 그러나 실 트래픽에서의 차이는 직접 측정 권장

## 6. 안전 및 정렬 (Safety & Alignment)

- Opus 4.6과 **유사한 안전 프로필** — 기만, 아첨, 오용 협력에서 낮은 비율
- **개선**: 정직성, 프롬프트 인젝션 저항
- **약간 약화**: 규제 물질에 대한 과도하게 상세한 해로움 감소 조언
- 정렬 평가 결론: **"대체로 잘 정렬되고 신뢰할 수 있으나, 완전히 이상적이지는 않다"**
- **Mythos Preview**가 여전히 가장 잘 정렬된 모델
- 전체 안전 평가는 [Claude Opus 4.7 System Card](https://anthropic.com/claude-opus-4-7-system-card)에서 확인 가능

## Soo에게 의미 있는 이유

1. **에이전트 자율성의 실질적 진전**: "밀착 감독이 필요했던 작업을 자신 있게 맡길 수 있다" — AInD 컨설팅에서 가장 설득력 있는 메시지. 고객사에 에이전트 도입을 제안할 때 **구체적 사례와 수치**를 인용 가능
2. **하네스 재조정 필요성**: 지시사항 문자 그대로 준수는 **기존 프롬프트/하네스가 깨질 수 있음**을 의미. Starforge 같은 하네스에서 모델 업그레이드 시 **자동 프롬프트 검증 파이프라인**의 필요성을 뒷받침
3. **`/ultrareview`와 Auto Mode**: 코딩 에이전트 워크플로우에서 **품질 게이트와 자율 실행의 균형** — AInD 컨설팅에서 "에이전트를 어떻게 신뢰할 것인가?"라는 질문에 대한 구체적 답변
4. **토크나이저 변경 주의**: 동일 입력이 1.35배까지 더 많은 토큰을 소비할 수 있다는 점은 **비용 관리 전략**에서 중요. 특히 대규모 에이전트 배포 시 비용 예측에 영향
5. **사이버 세이프가드 전략**: Mythos급 모델의 광범위한 출시를 향한 단계적 접근 — 프런티어 모델 접근 정책의 진화를 추적하는 데 중요 (지난주 "프런티어의 종말" 에세이와 연결)

## 벤치마크 주석

- GPT-5.4 및 Gemini 3.1 Pro: API에서 사용 가능한 최고 모델 버전과 비교
- SWE-bench Verified/Pro/Multilingual: 암기 징후가 있는 문제 제외해도 Opus 4.7의 개선 마진 유지
- Terminal-Bench 2.0: Terminus-2 하네스, thinking 비활성화, 1× 보장/3× 상한 리소스, 태스크당 5회 평균
- CyberGym: Opus 4.6 점수가 66.6에서 73.8로 업데이트 (하네스 파라미터 개선)
