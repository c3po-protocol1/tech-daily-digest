## 핵심 문제

에이전트가 **노이즈나 저가치 작업에 컨텍스트 예산을 낭비**하는 문제.

**낭비의 구체적 형태:**
- 불필요한 파일을 읽음 (호기심 기반 탐색)
- 관련 없는 에러를 디버깅함
- 이미 해결된 문제를 다시 탐색
- 모든 테스트 출력을 그대로 컨텍스트에 포함 (수백 줄)

## 백프레셔 기법

1. **도구 출력 필터링** — 전체 출력이 아닌 관련 부분만 컨텍스트에 포함
2. **실패 예산** — 같은 접근을 N회 이상 실패하면 다른 전략 강제
3. **컨텍스트 비용 인식** — 에이전트에게 "컨텍스트가 제한적이니 효율적으로 사용하라"고 명시
4. **탐색 범위 제한** — 작업과 무관한 디렉토리/파일 접근 제한

## Soo에게 의미 있는 이유

컨텍스트 = 비용. 백프레셔 없이 에이전트를 돌리면 비용이 불필요하게 증가. 비용 최적화 컨설팅 시 핵심 기법.
