## 핵심 문제: 컨텍스트 드리프트

장기 세션에서 초기 지시가 점점 희미해지는 현상. 에이전트가 원래 목표에서 서서히 벗어남.

**드리프트의 구체적 형태:**
- 코딩 스타일이 세션 중간에 변함
- 초기에 지시한 아키텍처 패턴을 잊고 다른 패턴 사용
- 완료 기준을 잊고 관련 없는 작업에 빠짐

## 해결 패턴

### 주기적 리마인더
- N번의 도구 호출마다 핵심 지시를 **자동으로 재주입**
- "당신의 현재 목표는 X입니다. 완료 기준은 Y입니다."

### 구조화된 체크포인트
- 일정 간격으로 에이전트에게 "현재 상태를 보고하라"고 요청
- 보고서가 원래 목표와 일치하는지 자동 확인
- 불일치 시 경고 또는 방향 수정

### 세션 재개 패턴
- 세션 중단 시 **상태 스냅샷**을 파일로 저장
- 새 세션 시작 시 스냅샷을 컨텍스트에 주입
- 구조화된 형식(JSON, 체크리스트)이 자유 형식보다 세션 간 전달에 유리

## Soo에게 의미 있는 이유

장기 코딩 작업에서 "에이전트가 방향을 잃는" 문제의 해결책. R2가 장기 작업을 할 때 적용 가능.
