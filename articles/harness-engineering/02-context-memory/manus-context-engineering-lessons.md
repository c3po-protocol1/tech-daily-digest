## 배경

Manus(AI 에이전트 제품)를 만들면서 얻은 **실전 컨텍스트 엔지니어링 교훈**. Anthropic의 이론적 가이드를 실제 프로덕션에서 적용한 경험.

## KV-Cache 지역성 (Locality)

**기술적 배경:** LLM 추론 시 프롬프트의 접두사(prefix)가 같으면 이전 계산을 **KV-cache에서 재사용**. 이는 비용과 지연을 크게 줄임.

**실전 의미:**
- 시스템 프롬프트와 도구 정의는 **매 턴 동일하게** 유지
- 매번 도구 목록을 다르게 구성하면 → 캐시 미스 → 전체 재계산
- 프롬프트의 앞부분(접두사)은 가능한 **안정적으로** 유지

**비용 절감 효과:** 프로덕션에서 KV-cache 히트율을 높이면 추론 비용이 **50% 이상** 절감될 수 있음.

## 도구 마스킹 (Tool Masking)

**문제:** 에이전트에 20개 도구를 노출하면, 현재 상태에서 사용 불가능한 도구까지 "고려"하느라 혼란.

**해결:** 에이전트의 현재 상태에 따라 **사용 가능한 도구만 동적으로 노출**.

**핵심 기법 — KV-cache를 깨지 않는 마스킹:**
- 도구 목록 자체는 항상 같게 유지 (캐시 히트를 위해)
- 대신 **도구의 description에 "현재 사용 불가"를 표시**하거나
- 도구 호출 결과에서 **"이 도구는 현재 상태에서 사용할 수 없습니다"**를 반환
- 이렇게 하면 프롬프트 접두사는 변하지 않으면서 도구 사용을 제어 가능

## 파일시스템 메모리

**문제:** 컨텍스트 윈도우의 크기는 유한하지만, 에이전트가 기억해야 할 정보는 계속 증가.

**해결:** **파일시스템을 외부 메모리로 활용.**

- 작업 중간 결과를 파일에 저장 → 컨텍스트에서 제거 → 필요할 때 다시 읽기
- "scratchpad" 파일에 메모, 체크리스트, 중간 분석 결과 저장
- 장기 프로젝트에서는 **구조화된 상태 파일** (JSON/YAML)로 진행 상황 관리

이것은 인간이 메모장을 사용하는 것과 같은 패턴. 모든 것을 머리(컨텍스트)에 담지 않고, 필요한 것만 메모(파일)에서 꺼내 본다.

## 유용한 실패를 컨텍스트에 유지

**반직관적 교훈:** 에이전트의 **실패한 시도도 가치 있는 정보**.

- 성공만 남기고 실패를 제거하면 → 에이전트가 같은 실패를 반복
- "이 접근은 실패했다. 이유: X. 다음에는 Y를 시도하라"를 남기면 → 학습 효과
- 단, 오래된 실패는 정리해야 함 (영원히 남기면 노이즈)

## Soo에게 의미 있는 이유

Anthropic의 이론을 **프로덕션에서 검증**한 사례. 특히 KV-cache 지역성과 도구 마스킹은 대부분의 가이드에서 다루지 않는 **인프라 수준의 최적화**로, AInD 컨설팅에서 "비용 최적화"를 다룰 때 핵심 기법이 된다.
