# Writing a Good CLAUDE.md

- **출처:** [HumanLayer](https://www.humanlayer.dev/blog/writing-a-good-claude-md)
- **태그:** `#HumanLayer` `#CLAUDEmd` `#AGENTSmd` `#ProjectSetup`

---

## 핵심: CLAUDE.md는 에이전트의 온보딩 문서

새 팀원(=에이전트)에게 프로젝트를 설명하듯 작성하라.

## 좋은 CLAUDE.md의 특징

1. **구체적** — "좋은 코드를 작성하라" ❌ → "모든 함수에 JSDoc 주석을 포함하라" ✅
2. **검증 가능** — 에이전트가 따랐는지 자동 확인 가능한 규칙
3. **안정적** — 매번 바뀌면 혼란. 프로젝트 전체에 일관되게 유지
4. **계층적** — 루트 CLAUDE.md (전체 규칙) + 디렉토리별 CLAUDE.md (로컬 규칙)

## 필수 포함 항목

- **빌드 명령어** — `npm run build`, `go test ./...` 등
- **테스트 명령어** — 에이전트가 자기 검증에 사용
- **린트 규칙** — 코드 스타일 자동 검증
- **프로젝트 구조** — 주요 디렉토리와 파일의 역할
- **금지 사항** — "이 파일은 수정하지 마라", "전역 상태 사용 금지"
- **참조 모듈** — "src/modules/auth를 참조하여 같은 패턴으로"

## 피해야 할 것

- 모순되는 지시
- 너무 많은 규칙 (에이전트가 모두 따르기 어려움)
- 암묵적 가정 ("당연히 알겠지"는 에이전트에게 통하지 않음)
- 버전이 안 맞는 정보 (코드는 바뀌었는데 CLAUDE.md는 이전 상태)

## Soo에게 의미 있는 이유

우리의 SPEC.md, ARCHITECTURE.md, PRINCIPLES.md 패턴이 바로 이 원칙을 따르고 있다. AInD 컨설팅에서 "프로젝트에 에이전트를 도입하려면 먼저 무엇을 준비해야 하는가"의 답.
