# Effective Harnesses for Long-Running Agents

- **출처:** [Anthropic](https://www.anthropic.com/engineering/effective-harnesses-for-long-running-agents)
- **태그:** `#Anthropic` `#LongRunning` `#ContextWindow` `#Handoff`

## 배경

수 시간~수 일에 걸친 장기 에이전트 작업에서, **각 세션이 이전 세션의 기억을 갖지 못하는 문제**를 해결한 Anthropic의 원조 harness 설계 글이다. Claude Agent SDK 기반.

## 핵심 문제: 장기 에이전트의 실패 패턴

Opus 4.5를 Claude Agent SDK에서 루프로 실행해도, 고수준 프롬프트("claude.ai 클론을 만들어라")만으로는 프로덕션 품질 앱을 만들지 못했다. 두 가지 실패 패턴이 반복됨:

### 실패 1: 한 번에 너무 많이 하려 함 (One-shotting)
- 에이전트가 전체 앱을 한 세션에 구현하려 시도
- 컨텍스트 윈도우 중간에 소진 → 기능이 반쯤 구현된 채 세션 종료
- 다음 세션이 반쯤 완성된 코드를 보고 추측해야 함 → 시간 낭비 + 회귀

### 실패 2: 조기 완료 선언
- 몇 가지 기능이 완성된 후, 에이전트가 주변을 살피고 "일이 끝났다"고 판단
- 실제로는 스펙의 30%만 완성된 상태에서 작업 중단

## 해결책: 2-에이전트 구조

### Initializer Agent (첫 세션에서 1회 실행)

역할: 고수준 프롬프트를 받아 **작업 환경을 완전히 설정**.

구체적 산출물:
- **Feature List** — 전체 구현을 독립적인 기능 단위로 분해한 체크리스트
  - 각 기능은 하나의 세션에서 완료 가능한 크기
  - 의존성 순서를 고려하여 정렬
  - 완료 여부를 표시할 수 있는 체크박스 형태
- **init.sh** — 프로젝트 스캐폴딩 스크립트
  - 디렉토리 구조 생성
  - 의존성 설치
  - 기본 설정 파일 생성
  - 빌드/테스트 파이프라인 설정
- **아키텍처 문서** — 기술 스택, 디렉토리 규칙, 코딩 스타일

### Coding Agent (매 세션마다 실행)

역할: Feature List에서 다음 미완성 기능을 선택하고 **한 세션에 하나의 기능만 구현**.

세션 내 동작:
1. Feature List 확인 → 다음 미완성 기능 선택
2. 해당 기능 구현
3. 자기 검증 (빌드 + 테스트 + 수동 확인)
4. Feature List에서 완료 표시
5. **핸드오프 아티팩트 작성** → 다음 세션을 위한 상태 기록

### 핸드오프 아티팩트의 구성

핵심은 "다음 에이전트가 이어받기 위해 알아야 할 모든 것":
- ✅ 완료된 기능 목록
- 🔧 현재 작업 중이었던 것 (있다면)
- ⚠️ 알려진 이슈/버그
- 📋 다음 세션에서 해야 할 일
- 🏗️ 코드 상태 요약 (어떤 파일이 변경되었는지)

### "깨끗한 상태(Clean State)"의 정의

매 세션 종료 시 코드는 **메인 브랜치에 머지해도 될 수준**이어야 함:
- 주요 버그 없음
- 코드가 정돈되고 문서화됨
- 다른 개발자(또는 에이전트)가 새 기능을 바로 시작할 수 있는 상태

## 실험 결과

이 harness를 적용했을 때:
- 단일 에이전트 대비 **기능 완성도 대폭 향상**
- 세션 간 회귀(이전에 작동하던 것이 깨지는 현상) 감소
- Feature List가 전체 진행 상황의 가시성 제공

## 한계와 남은 과제

- **Self-evaluation 문제** — 에이전트가 자기 코드를 "잘 했다"고 평가하는 경향 (다음 글에서 해결)
- **Context Anxiety** — Sonnet 4.5에서 컨텍스트 한계 접근 시 조기 마무리 경향
- **UI/디자인 품질** — 기능은 작동하지만 시각적 품질이 밋밋

## Soo에게 의미 있는 이유

이 글은 **우리가 Holocron에서 하려는 것과 직결**된다. R2가 Claude Code로 장기 작업을 할 때 정확히 이 패턴(Feature List + one-at-a-time + 핸드오프)을 따르고 있다. 또한 AInD 컨설팅에서 "에이전트가 장기 프로젝트를 어떻게 수행하는가"의 핵심 레퍼런스.
