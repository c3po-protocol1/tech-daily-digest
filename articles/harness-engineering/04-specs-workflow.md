## 4.1 AGENTS.md — 리포 로컬 에이전트 지시서 표준

**출처:** [github.com/agentsmd/agents.md](https://github.com/agentsmd/agents.md)

### 핵심 내용

AI 코딩 에이전트에게 **리포지토리 안에서 어떻게 작업해야 하는지** 알려주는 경량 오픈 포맷.

**왜 필요한가:**
- 에이전트가 프로젝트에 처음 투입될 때 "이 프로젝트는 어떤 규칙을 따르는가?"를 알아야 함
- 구두 설명이나 외부 문서가 아닌, **리포 안에 있는 기계 판독 가능한 지시서**

**주요 구성:**
- 빌드/테스트/린트 명령어
- 코딩 규칙과 스타일 가이드
- 디렉토리 구조 설명
- 금지 사항 (e.g., "이 파일 수정 금지")
- 선호하는 패턴과 참조 모듈

**Claude Code, Codex, Cursor 등에서 채택** — 사실상 업계 표준으로 자리잡는 중.

## 4.3 GitHub Spec Kit — 스펙 기반 개발 도구

**출처:** [github.com/github/spec-kit](https://github.com/github/spec-kit)

### 핵심 내용

GitHub이 만든 **스펙 기반 개발(Spec-Driven Development)** 도구. 에이전트가 명시적인 제품/엔지니어링 스펙에 맞춰 실행하도록 설계.

**스펙 기반 개발의 핵심:**
- 코딩 시작 전에 **무엇을 만들 것인지 명확히 정의**
- 스펙이 에이전트의 작업 범위를 제한하고 방향을 제시
- 스펙 대비 구현 상태를 **자동으로 검증 가능**

**Holocron과의 연관:** 우리가 SPEC.md → ARCHITECTURE.md → PRINCIPLES.md로 spec을 먼저 정의하고 R2에게 구현을 맡기는 패턴이 바로 이것.

## 4.4 Thoughtworks — Understanding Spec-Driven Development

**출처:** [martinfowler.com](https://martinfowler.com/articles/exploring-gen-ai/sdd-3-tools.html)

### 핵심 내용

강한 스펙이 AI 보조 소프트웨어 전달을 더 신뢰할 수 있게 만드는 이유.

**Kiro, spec-kit, Tessl 분석:**
- 각 도구가 "스펙 → 구현 → 검증" 파이프라인을 어떻게 다루는지 비교
- **스펙이 약하면 에이전트가 방향을 잃음** — 구현의 품질은 스펙의 품질에 비례
- 좋은 스펙의 특징: 구체적, 검증 가능, 우선순위 명확, 범위 한정

### 핵심 교훈
> "에이전트에게 좋은 코드를 기대하려면, 먼저 좋은 스펙을 작성하라."

## 4.5 HumanLayer — 12 Factor Agents

**출처:** [humanlayer.dev](https://www.humanlayer.dev/blog/12-factor-agents)

### 핵심 내용

Heroku의 12-Factor App에서 영감을 받은 **프로덕션 에이전트의 12가지 운영 원칙**.

**주요 원칙:**
1. **명시적 프롬프트** — 암묵적 가정 없이 모든 지시를 명시
2. **상태 소유권** — 에이전트의 상태를 누가 관리하는지 명확히
3. **깨끗한 일시정지/재개** — 중단되어도 이어서 작업 가능
4. **결정적 도구** — 도구 호출의 결과가 예측 가능
5. **관찰 가능성** — 에이전트가 무엇을 하고 있는지 추적 가능
6. **멱등성** — 같은 작업을 반복해도 부작용 없음
7. **스코프 제한** — 에이전트의 권한과 접근 범위 최소화
8. **버전 관리** — 프롬프트, 도구, 환경의 버전을 관리
9. **테스트 가능성** — 에이전트의 행동을 자동으로 검증
10. **로그 기반 디버깅** — 모든 결정과 행동을 로깅
11. **환경 분리** — dev/staging/prod 환경 분리
12. **graceful degradation** — 부분 실패에도 전체가 멈추지 않음

### 핵심 교훈
> "프로덕션 에이전트는 프로덕션 서비스다. 같은 수준의 운영 원칙이 필요하다."

## 4.6 12-Factor AgentOps

**출처:** [12factoragentops.com](https://www.12factoragentops.com/)

### 핵심 내용

12-Factor Agents의 **운영(Ops) 관점** 컴패니언. 컨텍스트 규율, 검증, 재현 가능한 에이전트 워크플로우에 초점.

- 에이전트 워크플로우의 **재현 가능성** 강조
- 같은 입력에 같은 결과를 내는 **결정적 파이프라인** 설계
- 에이전트 실행의 **감사(audit) 가능성** — 무엇을 왜 했는지 추적

### 핵심 교훈
> "에이전트도 CI/CD처럼 재현 가능하고 감사 가능해야 한다."
