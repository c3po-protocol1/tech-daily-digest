## 요약

2026년 3월 31일 NPM 레지스트리의 소스맵을 통해 Claude Code 전체 소스코드가 노출된 후, 한국 개발자가 Claude Code로 Claude Code를 분석한 상세 분석서.

### 핵심 구조 (약 1,884개 소스 파일)

- **Agentic Loop 6단계**: 메시지 입력 → 시스템 프롬프트 조립 → API 호출 → 권한 체크 → 도구 실행 → 결과 기반 반복
- **컨텍스트 조립**: 시스템 컨텍스트(git 상태) + 유저 컨텍스트(CLAUDE.md 4단계 계층 탐색)
- **서브에이전트(Task 도구)**: 격리된 대화와 제한된 도구셋으로 자체 루프, 완료 시 결과 반환
- **컴팩션**: 긴 대화는 주기적으로 오래된 메시지를 요약해 컨텍스트 윈도우 관리
- **maxResultSizeChars**: 도구 결과가 한도 초과 시 임시 파일로 저장, 경로만 전달

### 권한 모드

- **bypassPermissions**: 모든 체크 스킵
- **acceptEdits**: 파일 편집 자동 승인, bash는 확인 요청
- 읽기 전용 도구(Read, Glob, Grep)는 모든 모드에서 자동 승인

### 대화 저장/복원

- `~/.claude/`에 JSON 트랜스크립트 저장
- `--resume`으로 재개 시 전체 히스토리 로드 + CLAUDE.md 재탐색

## Soo에게 의미 있는 이유

AI 코딩 에이전트의 내부 구조를 이해하는 핵심 참고자료. AInD 컨설팅에서 "에이전트가 실제로 어떻게 작동하는가"를 설명할 때 구체적 아키텍처 레퍼런스로 활용 가능. 특히 Agentic Loop, 컨텍스트 관리, 서브에이전트 패턴은 에이전트 설계의 표준 패턴으로 자리잡고 있다.
