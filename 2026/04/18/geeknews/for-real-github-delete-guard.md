## 요약

테스트 레포를 지우려다 **9명짜리 GitHub 조직을 통째로 날린** 개발자가 만든 삭제 방지 크롬 익스텐션이다.

### 배경 스토리

- Discord 웹훅 봇 테스트 중 테스트 레포 Settings인 줄 알고 조직 Settings에서 삭제 실행
- 확인 텍스트를 읽지 않고 복사-붙여넣기로 통과
- 레포, 이슈, PR, 디스커션, CI 히스토리 전부 한 번에 삭제
- GitHub 지원팀 메일 후 **영업일 7일 대기**

### for-real의 작동 방식

- 등록된 레포/조직에서 삭제 버튼 클릭 시 가로채기
- **20자 랜덤 코드 직접 타이핑** 필요 (5분간 언락)
- **복사-붙여넣기 차단**
- 웹스토어 심사 중, 현재는 언팩 설치만 가능

### 핵심 문제 인식

GitHub의 조직 삭제가 레포 삭제와 동일한 UI/가드레일을 사용하지만, 파괴력은 100배 이상 차이. 복구 경로도 똑같이 "지원팀 메일"뿐이다.

## Soo에게 의미 있는 이유

개발 조직에서 실수로 인한 데이터 손실은 항상 발생할 수 있다. 특히 AI 에이전트가 GitHub 작업을 자동화하는 시대에 이런 **안전장치 도구**의 중요성이 더 커진다.
