## 요약

Anthropic의 Claude Opus 4.7 마이그레이션 가이드에서는 새 토크나이저가 "대략 1.0~1.35배 더 많은 토큰"을 사용한다고 명시했지만, 실제 기술 콘텐츠에서 측정한 결과 **1.47배**까지 증가하는 것으로 나타났다.

### 실측 데이터

**실제 Claude Code 콘텐츠 7개 샘플 테스트:**

| 콘텐츠 유형 | 4.6 토큰 | 4.7 토큰 | 비율 |
|---|---|---|---|
| CLAUDE.md (5KB) | 1,399 | 2,021 | **1.445x** |
| 사용자 프롬프트 | 1,122 | 1,541 | **1.373x** |
| 블로그 포스트 | 1,209 | 1,654 | **1.368x** |
| Git 커밋 로그 | 910 | 1,223 | **1.344x** |
| 터미널 출력 | 652 | 842 | **1.291x** |
| Python 스택 트레이스 | 1,736 | 2,170 | **1.250x** |
| 코드 diff | 1,226 | 1,486 | **1.212x** |

7개 샘플 가중 평균: **1.325x** (8,254 → 10,937 토큰)

### 토크나이저 변경 패턴 3가지

1. **CJK/이모지/기호**: 1.005~1.07x로 거의 변화 없음
2. **영어 및 코드**: 1.20~1.47x로 큰 증가. 영어 문자당 토큰이 4.33 → 3.60으로 하락
3. **코드가 산문보다 더 큰 영향**: 코드는 키워드, import, 식별자 등 반복 고빈도 문자열이 많아 BPE 병합이 더 크게 작용

### 왜 토큰이 더 많은 토크나이저를 출시했나?

Anthropic의 설명: "더 정확한 지시 따르기, 특히 낮은 노력 수준에서. 모델이 하나의 항목에 대한 지시를 다른 항목에 자동으로 일반화하지 않음"

더 작은 토큰은 개별 단어에 대한 어텐션을 강제하여 **더 정밀한 지시 따르기, 문자 수준 작업, 도구 호출 정밀도**를 높인다.

### IFEval 벤치마크 결과 (N=20)

- Strict prompt-level: 85% → **90%** (+5pp)
- Strict instruction-level: 86% → **90%** (+4pp)
- Loose 평가는 동일 (90%)

### 실제 비용 영향

80턴 세션 기준 Claude Code 사용 시 4.6에서 4.7로 전환 시 비용이 약 **17% 증가**한다. Max 사용자의 경우 퀘타 소진이 더 빨라지고, 캐시된 프리픽스 비용이 턴당 증가하며, 레이트 리밋에 더 빨리 도달하게 된다.

## Soo에게 의미 있는 이유

LLM 비용 최적화는 AInD 컨설팅에서 항상 클라이언트가 묻는 핵심 질문이다. 이 분석은 **토크나이저 변경만으로도 17% 비용 증가**가 발생할 수 있음을 보여주며, 모델 업그레이드 시 단순 성능 비교를 넘어 **토큰 경제학까지 고려해야 한다**는 중요한 교훈을 제공한다. 특히 CJK(한국어 포함)는 거의 영향이 없다는 점도 한국 시장에서 유용한 정보다.
