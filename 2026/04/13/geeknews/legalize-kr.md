## 요약

대한민국의 모든 법령과 판례를 **Markdown 파일 + Git 히스토리**로 관리하는 프로젝트. 법제처 국가법령정보 공동활용 OpenAPI에서 데이터를 가져와 구조화한다.

**핵심 특징:**
- 모든 법령은 YAML 메타데이터가 포함된 Markdown 파일
- 모든 개정은 실제 날짜를 가진 **Git 커밋** — `git log`로 개정 이력 확인
- `git diff`로 법령 개정 내용을 코드처럼 비교
- `grep`으로 전체 법령/판례 텍스트 검색

**법령 구조:**
- 3,048개 디렉토리: `kr/{법률명}/{법률|시행령|시행규칙}.md`
- 관련 법령이 하나의 디렉토리에 함께 관리

**판례 구조:**
- 8개 사건 종류(민사, 형사, 일반행정 등)별 분류
- `{사건종류}/{법원등급}/{사건번호}.md` 형태

**실용 예시:**
```bash
# 민법에서 "혼인" 관련 조문 찾기
grep -n "혼인" kr/민법/법률.md

# 민법 개정 이력
git log --format='%h %ad %s' --date=short -- kr/민법/법률.md

# 개정 내용 비교 (민법 제809조 동성동본 금혼 폐지)
git diff 로 실제 조문 변경 확인
```

## Soo에게 의미 있는 이유

"코드로서의 법률(Law as Code)" 개념의 한국판 구현. AI가 법률 문서를 분석하는 데 이상적인 포맷이다. RAG 파이프라인에 법령 Git repo를 연결하면 법률 AI 서비스의 기반이 될 수 있다. LegalTech + AI 컨설팅 기회의 실증 사례.
