# 관용적 디자인(Idiomatic Design)을 되돌려달라

> **출처**: [John Loeber](https://essays.johnloeber.com/p/4-bring-back-idiomatic-design) | [HN 토론](https://news.ycombinator.com/item?id=47738827)
> **점수**: 406점
> **태그**: `#UX` `#디자인` `#UI일관성` `#소프트웨어역사`

## 요약

저자 John Loeber는 데스크톱 소프트웨어 시대(Windows 95~7)의 **일관된 인터페이스 디자인**이 웹 시대에 와서 어떻게 붕괴되었는지를 분석한다.

**핵심 개념 - "디자인 관용구(Design Idiom)":**
체크박스, File/Edit/View 메뉴, 상태바 등은 사용자가 별도 학습 없이 사용할 수 있는 **공유된 디자인 패턴**이다. 데스크톱 시대에는 OS의 GUI 라이브러리가 이런 패턴을 강제해 생태계 전체에 걸친 **동질적 인터페이스(homogeneous interfaces)**를 만들었다.

**데스크톱 시대의 강점:**
- File/Edit/View 메뉴 구조가 표준 — Photoshop이든 Excel이든 동일
- ALT+F 같은 키보드 단축키가 보편적
- 모든 버튼이 명확히 라벨링되고, "이게 뭘 하는 거지?" 순간이 없음
- 밑줄로 단축키를 표시하는 것까지 일관됨

**브라우저 시대의 문제:**
- 날짜 선택, 카드 번호 입력 등 같은 작업에 수백 가지 다른 UI
- 앱마다 다른 키보드 단축키
- Figma와 Linear 같은 최고의 앱조차 완전히 다른 인터페이스
- 사용자는 매번 "내가 원하는 기능이 어디 있지?"를 반복해야 함

저자는 웹 시대에도 **디자인 관용구의 가치**를 되살려야 한다고 주장한다. 학습 비용이 제로인 인터페이스가 최고의 인터페이스라는 원칙을 재확인한다.

## Soo에게 의미 있는 이유

AI가 UI를 자동 생성하는 시대가 오면, 디자인 관용구의 중요성은 더욱 커진다. AI 코딩 에이전트가 생성하는 UI가 일관된 패턴을 따르지 않으면 사용자 경험이 급격히 저하된다. DESIGN.md 같은 AI 에이전트용 디자인 시스템 문서가 왜 필요한지를 설명하는 좋은 맥락.
