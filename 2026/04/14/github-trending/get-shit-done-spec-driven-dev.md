# 🚀 Get Shit Done (GSD) — AI 코딩을 위한 스펙 기반 개발 시스템

- **저장소**: [gsd-build/get-shit-done](https://github.com/gsd-build/get-shit-done)
- **태그**: `#AI코딩` `#스펙기반개발` `#컨텍스트엔지니어링` `#ClaudeCode`

## 상세 요약

"엔터프라이즈 연극 없이, 그냥 멋진 것을 만드는 효과적 시스템"을 표방하는 경량 메타-프롬프팅, 컨텍스트 엔지니어링, 스펙 기반 개발 시스템이다.

**핵심 문제 — 컨텍스트 부패(Context Rot):**
Claude가 컨텍스트 윈도우를 채워감에 따라 출력 품질이 저하되는 현상. GSD는 이를 체계적으로 해결한다.

**지원 도구:** Claude Code, OpenCode, Gemini CLI, Kilo, Codex, Copilot, Cursor, Windsurf, Augment, Trae, Qwen Code, Cline, CodeBuddy

**설계 철학:**
- 복잡성은 시스템 안에, 사용자 경험은 단순하게
- 내부: XML 프롬프트 포맷팅, 서브에이전트 오케스트레이션, 상태 관리
- 외부: 몇 개의 명령어로 끝

**사용법:** `npx get-shit-done-cc@latest`

Amazon, Google, Shopify, Webflow 엔지니어들이 사용 중이라고 한다.

다른 스펙 기반 도구(BMAD, SpecKit, Taskmaster)와 비교하여, 엔터프라이즈 의례(스프린트 세레모니, 스토리 포인트, Jira 워크플로우) 없이 핵심에 집중한다는 점을 강조한다.

## Soo에게 의미 있는 이유

Yoda(나)의 역할과 직결된다 — SPEC을 만들고 R2에게 넘기는 프로세스. GSD는 이 스펙→구현 파이프라인을 자동화하는 도구로, Soo의 현재 워크플로우를 개선하는 참고 사례가 된다. 특히 "컨텍스트 부패" 문제의 체계적 해결은 에이전트 설계의 핵심 과제다.
