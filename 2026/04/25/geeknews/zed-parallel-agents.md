## 상세 요약

Zed 에디터가 같은 창 안에서 여러 에이전트 스레드를 동시에 실행하고 조율할 수 있는 "병렬 에이전트(Parallel Agents)" 기능을 출시했다.

**새로운 Threads Sidebar**:
- 에디터 하단 좌측 아이콘 또는 `option-cmd-j`(macOS)로 접근
- 각 스레드의 폴더/저장소 접근 범위를 개별적으로 제어
- 여러 에이전트가 동시에 서로 다른 작업을 수행 가능

**핵심 개선사항**:
- 하나의 에디터 창에서 여러 AI 에이전트 스레드를 동시 실행
- 각 스레드가 작업할 수 있는 디렉토리/파일 범위를 격리
- 에이전트 간 작업이 충돌하지 않도록 범위(scope) 관리

이는 AI 코딩 워크플로에서 "한 번에 하나의 에이전트"라는 제약을 깨는 진화다. 프론트엔드 작업을 하는 에이전트와 백엔드 작업을 하는 에이전트를 동시에 돌릴 수 있게 되면, 개발 처리량이 크게 늘어날 수 있다.

최신 Zed 릴리스에서 바로 사용 가능하다.

## Soo에게 의미 있는 이유

IDE의 AI 통합이 "단일 에이전트 대화"에서 "병렬 에이전트 오케스트레이션"으로 진화하고 있다. Cursor, VS Code + Claude Code 등 경쟁 도구들의 병렬 에이전트 지원 현황과 비교하면 AInD 컨설팅에서 개발 도구 추천 시 유용한 참고 자료다.
