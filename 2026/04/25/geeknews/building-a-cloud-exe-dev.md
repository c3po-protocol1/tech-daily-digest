## 상세 요약

Tailscale 공동 창업자 David Crawshaw가 새로운 클라우드 플랫폼 exe.dev를 만들고 있다는 개인적 에세이다. 시리즈 A 자금 조달 발표와 함께 공개되었다.

**현재 클라우드의 근본적 문제점**:
1. **VM이 잘못된 형태**: CPU/메모리 리소스에 묶여 있어, CPU와 메모리와 디스크를 원하는 대로 조합할 수 없음. Linux VM은 다른 Linux의 cgroup에서 실행되는 프로세스일 뿐인데, 자유롭게 돌릴 수 없음.
2. **디스크가 느림**: 원격 블록 디바이스는 HDD 시대엔 합리적이었지만, SSD 시대엔 시크 타임이 10ms→20μs로 줄었는데 네트워크 지연이 그대로라 IOPS 오버헤드가 10%에서 10배 이상으로 증가. EC2에서 200k IOPS 설정에 월 $10k인데, MacBook은 500k IOPS.
3. **네트워킹 비용**: 표준 이그레스 비용이 일반 DC 대비 10배. 대규모 고객만 할인.
4. **PaaS의 한계**: 벤더 종속적이고, 일반 컴퓨터보다 더 제약적인 환경.

**exe.dev의 접근**:
- VM 리소스 격리 문제 해결: CPU와 메모리를 확보하면 원하는 만큼 VM 실행
- TLS 프록시와 인증 프록시 기본 제공
- 로컬 NVMe 디스크 + 비동기 블록 복제
- 글로벌 애니캐스트 네트워크
- 에이전트 시대에 더 많은 소프트웨어가 필요해지므로, 더 나은 클라우드가 필요하다는 논리

**에이전트 시대와의 연결**: 에이전트가 코드 작성을 쉽게 만들면 소프트웨어가 폭증(Jevons 역설). 클라우드의 불편함이 더 큰 고통이 됨. 에이전트도 AWS API의 복잡한 추상화와 씨름하느라 컨텍스트 윈도우를 낭비.

## Soo에게 의미 있는 이유

에이전트 시대의 인프라 변화를 이해하는 데 중요한 글이다. "에이전트가 더 많은 소프트웨어를 만들면, 클라우드 인프라의 근본적 재설계가 필요하다"는 주장은 AInD 컨설팅의 인프라 전략 관점에서 참고할 만하다. Tailscale 창업자의 새로운 도전이라는 점에서 신뢰도가 높다.
