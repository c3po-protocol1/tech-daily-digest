## 상세 요약

Y Combinator 대표 Garry Tan이 자신의 AI 에이전트 운용 경험을 바탕으로 제안한 "Skillify"라는 에이전트 품질 관리 방법론이다. LangChain이 1억 6천만 달러를 유치했지만 테스트 도구만 제공하고 "무엇을 어떤 순서로 테스트할 것인가"라는 워크플로가 부재하다는 문제의식에서 출발한다.

**Skillify의 핵심 개념**:
- 에이전트의 실패를 일회성 프롬프트 수정이 아닌, **마크다운 스킬 파일 + 결정론적 스크립트 + 자동화 테스트**로 구성된 영구적 구조물로 전환
- 대화 중 "skillify it"이라고 말하면 에이전트가 10단계 프로세스를 자동 수행

**Latent vs Deterministic 구분**:
- **Latent(잠재적)**: 판단이 필요한 작업 — LLM 추론 영역
- **Deterministic(결정론적)**: 정밀성이 필요한 작업 — 코드 실행 영역
- 타임존 계산이나 캘린더 검색처럼 코드가 즉시 답할 수 있는 일을 LLM이 "머릿속으로" 하다가 틀리는 것이 핵심 버그

**10단계 체크리스트**: SKILL.md 작성 → 결정론적 스크립트 작성 → 유닛 테스트(vitest) → 통합 테스트 → LLM 평가(LLM-as-judge) → 리졸버 트리거 등록 → 리졸버 평가 → 도달 가능성/중복 감사 → E2E 스모크 테스트 → 브레인 파일링 규칙

**실제 사례**:
- 10년 전 싱가포르 출장 일정: 에이전트가 5분간 API 호출 후 로컬 3,146개 파일에서 100ms 만에 찾을 수 있었던 데이터 발견
- "다음 회의 28분 후" vs 실제 88분 후: UTC→PT 변환을 LLM이 암산하다 정확히 1시간 오차

**핵심 인사이트**: 40개 이상의 스킬 운영 시 15%가 리졸버에 등록되지 않아 "어둠 속 기능"이 됨 → 에이전트 스킬의 발견 가능성(discoverability) 관리가 필수

## Soo에게 의미 있는 이유

Skillify는 OpenClaw의 스킬 시스템과 정확히 같은 문제를 다루고 있다. "실패 → 스킬 → 테스트 → 영구 수정" 워크플로는 AInD 컨설팅에서 에이전트 품질 관리 방법론으로 직접 활용할 수 있다. 특히 "latent vs deterministic" 구분은 에이전트 아키텍처 설계의 핵심 원칙이 될 수 있다.
