# Qwen3.5 모델 양자화 — 커뮤니티 버전 성능 저하 원인 규명

- **출처**: [Unsloth 분석](https://unsloth.ai)
- **토론**: [GeekNews](https://news.hada.io/topic?id=28658)
- **태그**: `#AI` `#LLM` `#quantization` `#Qwen` `#MLX`

## 상세 요약

AI 모델을 더 작고 빠르게 압축하는 '양자화(quantization)' 기술에서, 커뮤니티가 배포한 MLX 포맷의 Qwen3.5 모델이 심각한 문제를 보이고 있다는 분석이다.

**문제 현상:**
- 커뮤니티 양자화 버전에서 도구 호출(function calling) 오류 발생
- 무의미한 출력(gibberish) 생성
- 환각(hallucination) 현상 심화

**원인 (Unsloth의 분석):**
- AI 파인튜닝 도구 업체 Unsloth가 **150개 이상의 벤치마크 실험**을 통해 원인 규명
- **모델의 차원과 레이어가 균일하게 양자화되지 않았기 때문** — "이븐하게 구워지지 않았다"
- 이는 양자화에서 반복적으로 나타나는 패턴

**Unsloth의 해법:**
- 레이어별·차원별 차등 양자화 적용
- 모델 구조에 맞는 최적 양자화 전략 제시

**커뮤니티 반응:**
- Unsloth 창립자 Daniel Han에 대한 높은 평가: "진짜 천재"
- 오픈웨이트 모델이 나올 때마다 모델 구조부터 토크나이징 버그, 양자화 오류, 템플릿 오류까지 분석해서 공유하는 것에 감탄

## Soo에게 의미 있는 이유

로컬 LLM 활용이 늘어나면서 양자화 품질이 중요해지고 있다. "양자화하면 그냥 작아지는 것 아닌가?"가 아니라, 모델 구조에 맞는 정교한 양자화가 필요하다는 점은 AI 모델 배포 전략 수립 시 핵심 지식이다. 특히 Apple Silicon에서의 MLX 추론이 보편화되는 상황에서, 양자화 품질 검증은 실무적으로 중요한 과제다.
