## 요약

Google Cloud AI 디렉터이자 전 Chrome 엔지니어링 리더인 Addy Osmani가 JS Nation US 2026 인터뷰에서 AI 시대 시니어 개발자의 역할 변화를 논의.

**핵심 발언:**
- 현재 **90% 개발자가 AI로 코딩** — 기본 작업 방식이 됨
- 동시에 **신뢰도는 하락 중**: 복잡한 코드베이스, 엔터프라이즈 환경에서 AI 한계 인식 확산
- PR 크기가 **대폭 커지고 있음** — AI가 필요 이상으로 4배 많은 파일을 건드림

**Vibe Coding vs AI-Assisted Engineering 구분:**
- Vibe Coding: 탐색적, 프로토타입/MVP용, 코드 리뷰 덜 중요
- AI-Assisted Engineering: 아키텍처, 보안, 성능, 품질의 엔지니어링 리거가 중요

**Context Engineering의 부상:**
- Prompt Engineering → Context Engineering으로 진화
- 추가 데이터, 예시, 문서, 대화 이력 등 **인간 엔지니어가 가져올 맥락**을 에이전트에게 제공
- 구체적일수록 결과 품질 향상

**코드 리뷰의 중요성 급상승:**
- "코드를 읽는 것이 새로운 스킬이 아니라, 항상 그래왔던 스킬"
- 시니어는 이 능력을 잃지 말아야 하고, 주니어는 키워야 함
- 코드 리뷰가 주니어 교육의 핵심 포럼이 될 것

## Soo에게 의미 있는 이유

AInD 컨설팅에서 "AI 도구 도입 후 팀 구조와 프로세스를 어떻게 바꿀 것인가"에 대한 실질적 관점 제공. Context Engineering, Vibe Coding vs AI-Assisted Engineering 구분은 기업 교육 커리큘럼 설계에 직접 활용 가능.
