## 요약

개별 프로그래밍 언어의 문법보다 그 언어가 속한 **원형(ur-language) 패밀리**를 이해하는 것이 더 중요하다는 주장의 글. 같은 원형에 속한 언어 간 전환은 쉽지만, 다른 원형의 언어를 배우려면 새로운 신경 경로가 필요하다.

**7가지 원형 언어:**
1. **ALGOL** — 순차 실행, 조건문, 반복문, 함수. 대부분의 주류 언어(C, Python, Java, JS 등)가 여기 속함. Ada Lovelace까지 거슬러 올라가는 가장 오래된 원형
2. **Lisp** — 괄호 기반 접두사 표현식, 코드=데이터, 매크로 시스템으로 언어 의미론 자체를 재정의 가능. Common Lisp, Scheme, Clojure
3. **ML** — 일급 함수, Hindley-Milner 타입 시스템, 재귀 기반 반복, 패턴 매칭. Standard ML, OCaml, Haskell, Agda, Idris
4. **Self** — 객체 간 메시지 패싱으로 모든 동작 구현, 라이브 환경에서 개발. Smalltalk, Self. JavaScript는 Self의 클래스 없는 객체 시스템에서 파생
5. **Forth** — 스택 기반, 후위 표기법, 극도로 미니멀한 구조
6. **APL** — 배열 중심, 특수 기호로 된 연산자, 코드 자체가 패턴의 라벨
7. **Prolog** — 논리 프로그래밍, 사실과 규칙의 선언, 역추론

**핵심 통찰:** 시간이 지나며 원형들이 서로 영향을 주고받음. 1980년대 Self의 개념이 ALGOL 계열에 '클래스' 형태로 접목, 2010년 이후 ML의 아이디어(패턴 매칭, 타입 추론)가 ALGOL 계열에 등장.

## Soo에게 의미 있는 이유

AI 코딩 에이전트가 다양한 언어를 다루는 시대에, 언어의 **근본적 사고 패러다임** 차이를 이해하는 것이 핵심 역량. AInD 컨설팅에서 팀의 기술 스택 선택을 조언할 때 "어떤 원형에 익숙한가"가 학습 곡선을 결정한다.
