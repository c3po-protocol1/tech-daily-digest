# Microsoft Copilot이 약 150만개 GitHub PR에 광고 삽입

- **출처:** [GeekNews](https://news.hada.io/topic?id=28030)
- **태그:** `#GitHub` `#Copilot` `#Advertising` `#DevTools` `#Trust`

---

## 핵심 요약

GitHub Copilot이 자동 생성한 Pull Request 설명에 **"Raycast로 Copilot 코딩 에이전트 실행"** 등의 광고 문구를 삽입하는 사례가 다수 확인. 약 **150만 개의 PR**에 영향을 미친 것으로 추정.

### 왜 이것이 문제인가

- **개발 도구에 광고가 침투하는 전례 없는 사태** — 코드 리뷰의 맥락에 상업적 메시지가 섞임
- 개발자 도구에 대한 **신뢰(trust) 훼손** — AI 생성 콘텐츠가 중립적이지 않다는 증거
- PR 설명은 코드 변경의 맥락을 설명하는 **기술 문서** — 여기에 광고가 들어가는 것은 기술적 무결성을 훼손

### 더 큰 맥락

AI 서비스의 수익화 압박이 심화되면서 나타나는 현상. 무료/저가 AI 도구의 비즈니스 모델이 광고로 전환될 수 있다는 우려. 개발자 커뮤니티에서 **강한 반발** 발생.

## Soo에게 의미 있는 이유

AI 도구의 수익 모델과 사용자 신뢰의 균형 문제. AInD 컨설팅에서 "어떤 AI 도구를 기업에 추천해야 하는가"를 논할 때, 도구의 비즈니스 모델이 사용자 경험에 미치는 영향을 고려해야 함.
