## 핵심 요약

Shopify가 수백만 개의 비정형 커머스 데이터를 구조화하기 위해 **단순 LLM 호출에서 DSPy 기반 멀티 에이전트 아키텍처로 전환**한 과정.

### 전환 이유

단순 one-shot LLM 호출(GPT-4/5급)으로는:
- 비용이 과도 (수백만 건 처리)
- 복잡한 데이터 관계를 한 번에 파악하기 어려움
- 결과의 일관성 부족

### DSPy 기반 해결책

- **자체 호스팅 Qwen(32B/72B급)** 모델 + DSPy 최적화 조합
- GPT-4/5급 대형 모델 대신 작은 모델을 **자동으로 최적화**하여 비용 절감 + 정확도 유지
- 멀티 에이전트 구조: 데이터 추출, 검증, 정규화를 별도 에이전트가 분업

### 핵심 교훈

대규모 프로덕션 환경에서 에이전틱 AI를 실제로 적용한 귀중한 사례. **"가장 큰 모델을 쓰는 것이 항상 최선이 아니다"** — 작은 모델 + 좋은 파이프라인이 비용 대비 더 효과적일 수 있음을 증명.

## Soo에게 의미 있는 이유

AInD 컨설팅의 핵심 사례. 기업 고객이 "GPT-5를 쓰면 되지 않나?"라고 물을 때, Shopify 사례로 "작은 모델 + 좋은 아키텍처가 더 효과적이고 저렴할 수 있다"는 답변 가능. DSPy를 활용한 자동 최적화 파이프라인은 실무에 바로 참고 가능.
