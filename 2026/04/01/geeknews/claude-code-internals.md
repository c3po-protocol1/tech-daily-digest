## 핵심 요약

Claude Code가 터미널에서 어떻게 동작하는지 공식 문서와 유출 소스를 기준으로 핵심을 정리한 글.

### 에이전틱 루프 구조

"읽고 → 생각하고 → 도구 쓰고 → 결과 보고"라는 기본 루프:
1. 사용자 입력 수신
2. 컨텍스트 로딩 (CLAUDE.md, 프로젝트 파일, 스킬 등)
3. 모델 추론 (다음 행동 결정)
4. 도구 호출 (파일 읽기/쓰기, bash, grep 등)
5. 결과를 컨텍스트에 추가
6. 반복 (완료까지)

### 시스템 프롬프트 아키텍처

유출 소스에서 밝혀진 구조: 시스템 프롬프트가 단일 문자열이 아니라, **런타임에 조립되는 모듈식 캐시 가능 섹션**으로 구성. `SYSTEM_PROMPT_DYNAMIC_BOUNDARY` 마커로 정적 섹션(조직 간 캐시 가능)과 동적 섹션(사용자별 변경)을 분리.

### 사용자 승인 흐름

위험한 작업(파일 삭제, 외부 명령 실행 등)에 대한 승인 프로세스가 코드 레벨에서 어떻게 구현되어 있는지 분석.

## Soo에게 의미 있는 이유

AI 코딩 에이전트를 이해하거나 직접 구현하고자 하는 개발자에게 필수적인 참고 자료. 시스템 프롬프트의 모듈식 캐시 가능 설계는 harness 엔지니어링에서 KV-cache 최적화(Manus가 강조한 것)와 직접 연결.
