## 요약

크리에이트립(Creatrip)의 강병준 개발자가 사내 AI 코드 리뷰 시스템의 품질을 정량적으로 측정하고 개선한 과정을 상세하게 공유한 글이다.

**벤치마크 구축 (33점에서 시작):**
- Hotfix PR에서 원본 PR을 역추적하여, "AI 리뷰가 이 버그를 사전에 잡을 수 있었는가" 테스트 케이스 구성
- GPT 4o mini를 LLM-as-a-Judge로 활용한 자동 채점 시스템 구축
- 초기 점수 33점: "잘 검증해주고 있다는 느낌은 극소수 버그를 잡는 착각에 불과"

**서브에이전트의 함정:**
- 서브에이전트 팀 구성 시 탐지율이 오히려 하락, 비용 1.5~3배 증가
- 원인: 컨텍스트 압축에 의한 정보 손실, 관심사 제한에 의한 시야 축소, 교차 영역의 "책임 공백"
- 교훈: "여러 관점이 필요하면 서브에이전트가 아닌 독립 파이프라인에서 병렬 실행 후 결과 사후 병합"

**벤치마크 오염 발견:**
- AI가 벤치마크 데이터셋에 과적합. SWE-bench도 오염됨 (GPT 5.2, Claude Opus 4.5, Gemini 3 Flash가 정답 패치를 재현 가능)

**새로운 평가 체계 - 반영률(Adoption Rate):**
- adopted: AI 리뷰가 실제 코드 변경으로 이어진 경우
- engaged: 답글은 달렸지만 코드 변경 없음
- noised: 코드 변경도 답글도 없음

**A/B 테스트 결과:**
- GPT 5.2 Codex vs Opus 4.6: Codex가 더 꼼꼼하고 유효한 리뷰 제공
- Codex로 전환 후 주간 최고치 60% 반영률 달성

**최종 성과:** 질문 뱃지 도입, PR 본문에 Intent/Decisions 섹션 추가, 자동 스레드 resolve 등의 개선으로 **월간 반영률 63%** 달성

## Soo에게 의미 있는 이유

실제 프로덕션에서 AI 코드 리뷰를 운영하며 겪은 시행착오의 생생한 기록이다. 특히 "서브에이전트의 함정"과 "벤치마크 오염" 문제는 AI 시스템 운영에서 보편적으로 마주치는 과제로, AInD 컨설팅 시 반드시 공유해야 할 교훈들이다.
