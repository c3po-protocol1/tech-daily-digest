# Cal.diy - Cal.com의 오픈소스 커뮤니티 에디션

- **GitHub**: [calcom/cal.diy](https://github.com/calcom/cal.diy)
- **언어**: TypeScript / Next.js
- **오늘의 스타**: ⭐ 124/day (HN 124 points)
- **태그**: `#스케줄링` `#오픈소스` `#셀프호스팅` `#MIT`

## 요약

Cal.com이 최근 클로즈드 소스 전환을 선언한 가운데, 커뮤니티가 포크하여 만든 **100% MIT 라이선스** 오픈소스 스케줄링 플랫폼이다.

**Cal.com과의 차이점:**
- **엔터프라이즈 기능 없음**: Teams, Organizations, Insights, Workflows, SSO/SAML 등 EE 기능 제거
- **라이선스 키 불필요**: Cal.com 계정이나 라이선스 없이 모든 기능 사용 가능
- **100% 오픈소스**: "Open Core" 분리 없는 완전한 MIT 라이선스

기술 스택: Next.js, tRPC, React, Tailwind CSS, Prisma, Daily.co

셀프 호스팅 전용으로, 관리형 서비스는 제공하지 않는다. Cal.com의 상업적 용도에는 cal.com/sales를 이용하도록 안내하고 있다.

## Soo에게 의미 있는 이유

"오픈소스 → 클로즈드 소스" 전환에 대한 커뮤니티 포크 사례로, AI 도구 생태계에서도 유사한 패턴이 반복될 수 있다. 오픈소스 지속 가능성에 대한 논의의 좋은 사례다.
