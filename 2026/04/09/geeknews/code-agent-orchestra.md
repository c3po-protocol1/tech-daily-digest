## 상세 요약

Addy Osmani가 O'Reilly AI CodeCon에서 발표한 멀티 에이전트 코딩 오케스트레이션 패턴 종합 가이드.

### 패러다임 전환: 지휘자 → 오케스트레이터
- **지휘자 모델:** 하나의 에이전트, 동기식, 컨텍스트 윈도우가 한계
- **오케스트레이터 모델:** 여러 에이전트가 각자의 컨텍스트 윈도우로 비동기 작업. 개발자는 계획·배정·검수

### 단일 에이전트의 3가지 벽
1. **컨텍스트 과부하:** 대규모 코드베이스가 단일 윈도우 초과
2. **전문화 부재:** 만능 에이전트는 어디에도 능숙하지 않음
3. **조율 부재:** 에이전트 간 통신·태스크리스트·의존성 해결 불가

### 멀티에이전트의 4가지 복합 이점
1. **병렬성 (3x 처리량):** 프론트엔드·백엔드·테스트 동시 작업
2. **전문화:** db.js만 아는 에이전트가 범용 에이전트보다 DB 코드를 잘 씀
3. **격리:** Git worktree로 각 에이전트 별도 작업 디렉토리
4. **복합 학습:** AGENTS.md에 패턴과 교훈 축적 → 세션마다 개선

### AI 지원 코딩 8단계 모델 (Steve Yegge)
대부분 개발자가 Level 3-4에 머물러 있음. 오케스트레이션은 Level 6부터 시작하며, Level 5까지와는 근본적으로 다른 스킬셋 필요.

## Soo에게 의미 있는 이유
우리의 멀티에이전트 시스템(C-3PO/R2-D2/Yoda)이 바로 이 오케스트레이터 모델. AGENTS.md 기반 복합 학습은 이미 실천 중인 패턴이고, 이 발표가 체계적 프레임워크를 제공.
