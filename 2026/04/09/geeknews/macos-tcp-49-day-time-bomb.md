## 상세 요약

macOS XNU 커널에서 **정확히 49일 17시간 2분 47초** 연속 가동 후 TCP 네트워킹이 완전히 마비되는 버그가 발견되었다.

### 원인
커널 내부 TCP 타임스탬프 카운터가 32비트 밀리초 단위로 동작하는데, 이 값이 오버플로우되면서 TCP 연결 수립이 불가능해진다. 정확한 시간은 2^32 밀리초 = 49일 17시간 2분 47.296초.

### 증상
- 정확히 해당 시간이 지나면 새로운 TCP 연결 수립 불가
- 기존 연결은 유지되지만 새 연결은 모두 실패
- 재부팅 외에 해결 방법 없음

### 역사적 맥락
이 버그는 Windows 95의 유명한 49.7일 크래시 버그와 동일한 패턴(32비트 밀리초 카운터 오버플로우). 30년이 지난 후 macOS에서 같은 실수가 반복된 것.

### 영향
서버 환경에서 macOS를 49일 이상 연속 가동하는 경우 영향. 개인 사용자는 보통 그 전에 재부팅/업데이트하므로 발견이 늦었다.
