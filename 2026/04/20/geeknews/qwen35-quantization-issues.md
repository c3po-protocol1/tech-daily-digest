## 요약

AI 모델 양자화(quantization) 과정에서 커뮤니티가 배포한 MLX 포맷 Qwen3.5 모델들이 도구 호출 오류, 무의미한 토큰 생성 등 심각한 품질 문제를 보이고 있다는 분석.

**핵심 문제:**
- 커뮤니티 양자화 버전에서 tool call 오류 빈발
- 무의미한 토큰(garbage tokens) 생성
- 원본 모델 대비 현저한 성능 저하
- MLX 포맷 변환 과정에서의 정보 손실이 주 원인으로 추정

**배경:** 양자화는 모델을 더 작고 빠르게 압축하는 기술로, FP16 → Q4/Q8 등으로 비트 수를 줄여 메모리 사용량을 대폭 감소시킨다. 하지만 잘못된 양자화는 모델 품질을 심각하게 훼손할 수 있다.

**시사점:** 공식 양자화 버전과 커뮤니티 양자화 버전의 품질 차이가 크므로, 로컬 LLM 배포 시 양자화 출처를 신중히 선택해야 한다.

## Soo에게 의미 있는 이유

로컬 AI 모델 도입 시 양자화 품질이 실사용 성능에 미치는 영향을 보여주는 사례. "작은 모델을 빠르게" 전략의 함정을 이해하는 데 도움.
