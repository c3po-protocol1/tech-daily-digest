# rtk: LLM 토큰 소비를 60~90% 줄여주는 CLI 프록시

- **GeekNews**: [https://news.hada.io/topic?id=28245](https://news.hada.io/topic?id=28245)
- **태그**: `#LLM` `#토큰최적화` `#Rust` `#CLI` `#개발도구`

## 요약

AI 코딩 도구가 실행하는 CLI 명령어 출력을 LLM에 전달하기 전에 **필터링·압축하여 토큰을 60~90% 절감**하는 단일 Rust 바이너리(윈/맥/리눅스) 도구다.

### 핵심 기능

- **100개 이상의 명령어 지원**: git, grep, ls, cargo test 등 개발자가 자주 사용하는 CLI 명령어 출력을 지능적으로 압축
- **명령어 인식 필터링**: 각 명령어의 특성에 맞게 불필요한 출력(반복적 로그, 장황한 에러 메시지 등)을 제거
- **LLM 컨텍스트 최적화**: 압축된 출력만 LLM에 전달하여 컨텍스트 윈도우를 효율적으로 활용
- **크로스 플랫폼**: Rust로 작성되어 Windows, macOS, Linux 모두 지원

### 동작 원리

AI 코딩 에이전트가 터미널 명령을 실행할 때, 그 출력을 rtk가 가로채서 불필요한 정보를 제거하고 핵심 내용만 LLM에 전달한다. 예를 들어 `cargo test`의 수백 줄 출력에서 실패한 테스트와 에러 메시지만 추출한다.

## Soo에게 의미 있는 이유

AI 코딩 에이전트의 비용 효율성은 실무 도입의 핵심 장벽 중 하나다. 토큰 소비를 60~90% 줄일 수 있다면 API 비용이 크게 감소하며, 이는 AInD 컨설팅에서 "AI 도입의 ROI"를 설명할 때 중요한 사례가 된다. Caveman(아래 참조)과 함께 토큰 최적화 전략의 양대 축.
