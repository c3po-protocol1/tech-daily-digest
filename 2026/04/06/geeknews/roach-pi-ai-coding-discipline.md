## 요약

Claude Code 소스코드 유출 사건을 계기로 "AI 코딩 에이전트 내부에서 어떤 프롬프트가 주입되고 어떤 동작이 일어나는지 사용자가 전혀 알 수 없다"는 문제의식에서 탄생한 프로젝트다.

### 핵심 기능

- **완전한 투명성**: 모든 도구, 이벤트 훅, 스킬 주입, 에이전트 프롬프트가 TypeScript와 Markdown으로 구성되어 있어 소스를 직접 읽고 에이전트의 정확한 동작을 확인 가능
- **프롬프트 캐시 히트율 실시간 표시**: 푸터에서 컨텍스트 활용도를 세션별로 관찰
- **명령어 체계**:
  - `/clarify`: 에이전트가 동적으로 문맥 인식 질문을 생성하여 모호함 해소, 서브에이전트로 코드베이스 탐색 병행
  - `/plan`: 엄격한 실행 가능 구현 계획 생성 (플레이스홀더 없음)
  - `/ultraplan`: 5가지 리뷰어 관점(타당성, 아키텍처, 리스크, 의존성, 사용자 가치)을 서브에이전트로 병렬 수행 후 마일스톤 DAG로 종합
  - `/loop`: 정기적 프롬프트 스케줄링 (크론 스타일)
- **서브에이전트 시스템**: Single(일회성), Parallel(최대 8개 병렬), Chain(순차 파이프라인) 3가지 모드
- **세션 루프 스케줄러**: 최대 100개 동시 작업, 작업별 에러 격리, AbortController 기반 협력적 취소

### 설계 원칙

pi 코딩 에이전트(badlogic/pi-mono)의 확장으로 구현되며, 하드코딩된 템플릿이나 고정 질문 세트 없이 에이전트가 동적으로 질문을 생성하고 리뷰어를 선택한다.

## Soo에게 의미 있는 이유

AI 코딩 에이전트의 "블랙박스 문제"에 대한 직접적인 해결책이다. AInD 컨설팅에서 "AI 에이전트를 실무에 도입할 때 투명성과 제어 가능성이 왜 중요한가"를 설명하는 좋은 사례. 특히 `/ultraplan`의 다관점 병렬 리뷰 패턴은 엔터프라이즈 코딩 워크플로에 적용 가능한 아키텍처다.
