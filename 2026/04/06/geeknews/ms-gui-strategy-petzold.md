## 요약

🔥 **HN에서도 동시 트렌딩** (502 points)

PowerShell 창시자 Jeffrey Snover가 Microsoft의 30년간의 GUI 프레임워크 혼란을 분석한 글. 1988년 Petzold의 *Programming Windows*가 제시했던 "하나의 명확한 답"(Win32 API) 이후로, MFC → COM → WPF → Silverlight → Metro/UWP → WinUI 3 → MAUI까지 끊임없는 전환이 반복되었다.

모든 실패의 근본 원인은 기술적 문제가 아니라 **조직적 실패**(Windows 팀 vs .NET 팀의 13년 내전), 컨퍼런스 키노트 기반의 섣부른 플랫폼 선언, 개발자에게 사전 경고 없는 전략 전환이었다. 현재 Windows에는 17개의 GUI 기술이 공존하며, 가장 널리 사용되는 Electron은 정작 Microsoft가 만든 것이 아니다.
