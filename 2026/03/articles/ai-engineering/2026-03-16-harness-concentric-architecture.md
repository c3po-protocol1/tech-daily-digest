## Layer-by-Layer 상세 설명

### 🤖 CENTER: Coding Agent (코딩 에이전트)

```
                    ┌─────────────────────┐
                    │    🤖 CODEX AGENT   │
                    │                     │
                    │  Input:  프롬프트   │
                    │  Output: 코드, 테스트│
                    │          문서, CI    │
                    │                     │
                    │  특성:              │
                    │  • 단일 실행 ~6시간 │
                    │  • 병렬 실행 가능   │
                    │  • 자체 검증 수행   │
                    │  • PR 단위로 출력   │
                    └─────────────────────┘
```

- 에이전트 **자체는 범용 LLM** — 특별하지 않음
- 에이전트의 효과는 **전적으로 주변 하네스에 의존**
- 하네스 없는 에이전트 = 안장 없는 말. 빠르지만 방향 없음

- 에이전트가 구조를 모를 때 → architecture/ 맵을 참조

- 성능 문제 시 → PromQL로 메트릭 확인 → 최적화 코드 생성

## 전체 데이터 플로우 (Agent-Centric View)

```
                         👤 Human
                           │
              ┌────────────┴────────────┐
              │  목표 설정, 문제 분해,   │
              │  골든 원칙, 프롬프트     │
              └────────────┬────────────┘
                           │
                           ▼
              ┌─────── LAYER 4 ────────┐
              │  ♻️ 유지보수 에이전트들  │◀──── 편차 감지 시 자동 PR
              │                         │
              │  ┌──── LAYER 3 ─────┐  │
              │  │ 🏗️ 아키텍처 강제  │◀──── 규칙 위반 시 에러+수정 지침
              │  │                   │  │
              │  │ ┌── LAYER 2 ──┐  │  │
              │  │ │ 🔧 도구&환경 │◀──── worktree, CDP, 관측 스택
              │  │ │              │  │  │
              │  │ │ ┌ LAYER 1 ┐ │  │  │
              │  │ │ │📚 지식  │ │  │  │◀──── AGENTS.md + docs/
              │  │ │ │         │ │  │  │
              │  │ │ │ ┌─────┐ │ │  │  │
              │  │ │ │ │ 🤖  │ │ │  │  │
              │  │ │ │ │Agent│ │ │  │  │
              │  │ │ │ └──┬──┘ │ │  │  │
              │  │ │ │    │    │ │  │  │
              │  │ │ └────┼────┘ │  │  │
              │  │ │      │      │  │  │
              │  │ └──────┼──────┘  │  │
              │  │        │         │  │
              │  └────────┼─────────┘  │
              │           │            │
              └───────────┼────────────┘
                          │
                          ▼
                    ┌───────────┐
                    │  PR/Code  │──▶ Repository
                    │  Output   │
                    └───────────┘
```

**읽는 방법:**
- 에이전트는 중심에서 **바깥으로 출력**을 생산 (코드, PR)
- 하네스는 **바깥에서 안쪽으로** 에이전트를 감싸고 제어
- 인간은 **가장 바깥에서** 하네스를 설계하고 조율
- 각 계층이 건강해야 안쪽의 에이전트가 효과적으로 작동
