# 충분히 상세한 명세는 코드다

**출처:** https://news.hada.io/topic?id=27662

에이전틱 코딩에서 '명세 문서로 코드를 대체할 수 있다'는 주장에 대해, 명세가 충분히 정밀해지면 결국 코드와 동일한 형태로 수렴할 수밖에 없다는 근본적 한계를 지적한다. OpenAI의 Symphony 프로젝트를 사례로 분석한다.

**왜 중요한가:** AI 코딩의 한계와 가능성을 이해하는 데 중요한 철학적 논의로, 프롬프트 엔지니어링의 본질을 되묻는다.
