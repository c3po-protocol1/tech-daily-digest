# 내 작업 흐름을 A/B 테스트하지 말아 주세요

- **출처:** [GeekNews](https://news.hada.io/topic?id=27509)

Claude Code의 기능 변경 실험이 사용자의 작업 효율을 저하시켰다는 불만이 제기되었다. Anthropic의 A/B 테스트가 핵심 기능인 'plan mode'의 동작을 바꿔 워크플로우 품질을 떨어뜨렸다는 지적이다. 월 200달러를 지불하는 전문 사용자에게 사전 고지 없이 핵심 기능을 변경하는 것에 대한 윤리적 논의가 이어지고 있다.

**왜 중요한가:** AI 코딩 도구가 핵심 워크플로우에 깊이 통합될수록, 예고 없는 변경이 미치는 영향이 커진다. 도구 의존성 관리가 중요하다.
