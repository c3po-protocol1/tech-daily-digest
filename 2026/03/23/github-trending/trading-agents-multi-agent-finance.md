## 핵심 요약

여러 AI 에이전트가 각각 다른 역할을 맡아 금융 시장을 분석하고 거래를 실행하는 멀티 에이전트 트레이딩 프레임워크가 하루 1,108 스타를 기록하며 주목받고 있다. 중국어 포크 버전(TradingAgents-CN)도 동시에 215 스타/일로 트렌딩 중이어서, 글로벌하게 큰 관심을 끌고 있다.

TradingAgents의 아키텍처는 실제 헤지펀드의 투자팀 구조를 모방한다. **리서치 에이전트**가 뉴스, 실적 보고서, 거시경제 지표 등을 수집하고 분석한다. **테크니컬 에이전트**가 차트 패턴, 기술적 지표(RSI, MACD, 볼린저 밴드 등)를 분석한다. **리스크 에이전트**가 포트폴리오의 위험 노출도를 평가하고 포지션 크기를 조절한다. **트레이딩 에이전트**가 최종 매매 결정을 내리고 실행한다. 이 에이전트들은 **토론 프로토콜**을 통해 서로의 분석을 공유하고 합의에 도달한다.

멀티 에이전트 접근의 장점은 **"집단 지성(collective intelligence)"** 효과이다. 단일 에이전트가 모든 측면을 분석하면 컨텍스트 윈도우의 한계와 주의력 분산으로 인해 품질이 저하된다. 각 에이전트가 전문 영역에 집중하고 결과를 종합하면, 더 깊고 다각적인 분석이 가능하다.

기술적으로 LangGraph 기반으로 에이전트 간 워크플로를 정의하며, 각 에이전트는 독립적으로 도구(API 호출, 데이터 분석, 차트 생성 등)를 사용할 수 있다. 백테스팅 프레임워크도 내장되어 있어, 과거 데이터에 대한 전략의 성과를 검증할 수 있다.

이 프로젝트는 **멀티 에이전트 시스템의 실전 적용 사례**로서 의미가 크다. "에이전트 하나로 모든 걸 하려 하지 말고, 전문 에이전트들의 협업 시스템을 설계하라"는 원칙의 구체적 구현이며, 금융을 넘어 다른 도메인(의료 진단, 법률 분석, 기업 전략 등)에도 적용 가능한 아키텍처 패턴을 보여준다.

## Soo에게 의미 있는 이유

멀티 에이전트 아키텍처는 AInD 컨설팅의 고급 시나리오이다. 단일 에이전트를 넘어 여러 전문 에이전트의 협업 시스템을 설계하는 패턴은, 기업의 복잡한 비즈니스 프로세스를 AI로 자동화할 때 핵심이 된다.
