## 핵심 요약

Anthropic 내부에서 Claude Code의 Skills 기능을 실전 운용하며 축적한 노하우를 공유한 글이 GeekNews에서 79포인트라는 높은 점수를 기록했다. Skills는 Claude Code의 가장 활발히 사용되는 확장 포인트 중 하나로, **에이전트의 행동을 도메인별로 특화시키는 핵심 메커니즘**이다.

Skills의 본질은 **구조화된 지식을 에이전트의 컨텍스트에 주입하는 것**이다. 단순한 프롬프트 템플릿을 넘어, 에이전트가 특정 도메인의 태스크를 수행할 때 필요한 도구 사용법, 워크플로, 제약 조건, 모범 사례 등을 체계적으로 제공한다. 예를 들어 "GitHub 코드 리뷰" 스킬은 PR을 분석하는 방법, 코멘트를 작성하는 형식, 승인/거부 기준 등을 포함한다.

Anthropic이 수백 개의 스킬을 운용하면서 발견한 핵심 패턴들이 있다. 첫째, **스킬의 원자성(atomicity)**이다. 하나의 스킬이 너무 많은 것을 다루면 에이전트가 혼란스러워한다. "GitHub" 스킬 하나보다 "GitHub PR 리뷰", "GitHub 이슈 트리아지", "GitHub CI 디버깅" 등 세분화된 스킬이 더 효과적이다. 둘째, **스킬 간 충돌 관리**이다. 여러 스킬이 동시에 활성화될 때 상충되는 지시가 있으면 에이전트의 행동이 예측 불가능해진다. 이를 방지하기 위해 스킬 우선순위 체계와 명시적 충돌 해결 규칙이 필요하다.

셋째, **스킬의 버전 관리**이다. 스킬은 코드처럼 진화하며, 이전 버전과의 호환성, 변경 이력 추적, 롤백 메커니즘 등이 필요하다. 넷째, **스킬의 테스트**이다. 스킬이 의도한 대로 에이전트의 행동을 변화시키는지 검증하는 체계적인 테스트 방법론이 아직 충분히 발달하지 않았다. Anthropic은 "스킬 벤치마크" — 특정 스킬이 활성화된 상태에서 에이전트의 행동을 평가하는 태스크 세트 — 를 개발하여 이 문제를 해결하고 있다.

다섯째, **컨텍스트 윈도우 관리**이다. 스킬이 많아질수록 에이전트의 컨텍스트 윈도우를 차지하는 비율이 커지며, 이는 실제 태스크에 사용할 수 있는 컨텍스트 공간을 줄인다. 필요한 스킬만 동적으로 로딩하는 "lazy loading" 패턴과, 스킬 텍스트를 압축하는 기법이 중요하다.

이 글은 에이전트 하네스 엔지니어링의 구체적이고 실전적인 방법론을 제시하며, "하네스가 성능을 결정한다"는 원칙의 구체적 실현 방법을 보여준다.

## Soo에게 의미 있는 이유

하네스 엔지니어링의 실전 노하우는 AInD 컨설팅의 핵심 역량이 될 것이다. Anthropic이 내부에서 실제로 사용하는 스킬 운용 패턴은 기업에 에이전트를 도입할 때 직접 적용 가능한 방법론이다.
