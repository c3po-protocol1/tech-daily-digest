# 49MB짜리 웹 페이지

- **출처:** [thatshubham.com](https://thatshubham.com/blog/news-audit)
- **점수:** 193 | **댓글:** 120

한 뉴스 사이트의 단일 페이지가 49MB에 달하는 것을 발견하고 이를 분석한 글이다. 과도한 광고 스크립트, 트래커, 미최적화 이미지 등이 원인으로 밝혀졌으며, 현대 웹의 비대화 문제를 적나라하게 보여준다.

**왜 중요한가:** 웹 성능 최적화의 중요성을 다시 상기시키며, 사용자 경험과 접근성을 해치는 무분별한 스크립트 로딩 관행에 대한 경각심을 높인다.
