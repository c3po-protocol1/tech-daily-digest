## 핵심 요약

실제 API 트래픽을 캡처하여 **자동으로 테스트 케이스와 데이터 스텁을 생성**하는 오픈소스 도구. 18점으로 GeekNews 1위.

### 작동 방식

Keploy는 eBPF(extended Berkeley Packet Filter)를 활용하여 애플리케이션의 네트워크 트래픽을 **커널 레벨에서 캡처**. 코드 수정 없이 사이드카 방식으로 동작.

1. 애플리케이션에 Keploy를 연결
2. 실제 API 요청/응답을 자동 녹화
3. 녹화된 트래픽을 **재사용 가능한 테스트 케이스**로 변환
4. 외부 의존성(DB, 다른 서비스)을 **자동 스텁**으로 대체

### 핵심 가치

- **코드 수정 없이** 테스트 생성 (eBPF 기반 비침습적 방식)
- 실제 프로덕션/스테이징 트래픽 기반 → **현실적 테스트 데이터**
- 단위 테스트 + 통합 테스트 모두 자동 생성
- 외부 의존성 스텁 자동 생성으로 **격리된 테스트 환경** 구축
- 1,260 커밋, 활발한 개발 중

### AI 코딩과의 시너지

AI 에이전트가 생성한 코드의 **검증을 자동화**하는 데 매우 유용:
- 에이전트가 API를 만들면 → Keploy가 실제 트래픽에서 테스트를 자동 생성
- Harness 엔지니어링의 "자기 검증 루프"를 더 강력하게 구축 가능
- Anthropic의 "Evaluator가 Playwright로 실제 앱을 테스트" 패턴과 보완적

## Soo에게 의미 있는 이유

AI 에이전트가 생성한 API 코드의 테스트를 **사람 개입 없이** 자동 생성/실행할 수 있다면, harness의 검증 루프가 크게 강화. AInD 컨설팅에서 "테스트 자동화"를 논할 때 구체적 도구 추천.
